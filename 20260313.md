

## Desenvolvimento Web: Document Object Model (DOM)

O **DOM** é uma interface de programação para documentos HTML e XML. Ele representa a página de forma que os programas (como o JavaScript) possam alterar a estrutura, o estilo e o conteúdo do documento.

---

### 1. O que é o DOM?

Quando o navegador carrega uma página HTML, ele cria um modelo de objetos daquela página. O DOM não é o código HTML em si, mas uma representação viva dele na memória do navegador.

* **Conceito:** Cada tag HTML se torna um **Nó (Node)** na árvore.
* **Hierarquia:** Existe uma relação de Pai, Filho e Irmão entre os elementos.

---

### 2. A Árvore de Elementos

A estrutura começa no objeto global `window`, passa pelo `document` e chega na raiz do HTML.

* **Document:** O ponto de entrada para o conteúdo da página.
* **Root Element:** A tag `<html>`.
* **Parent Node:** Um elemento que contém outros (ex: `<body>` é pai de `<h1>`).
* **Child Node:** Um elemento contido em outro (ex: `<li>` é filho de `<ul>`).

---

### 3. Métodos de Seleção de Elementos

Para manipular um elemento com JavaScript, primeiro precisamos "capturá-lo". Os métodos mais comuns são:

| Método | Descrição | Exemplo |
| --- | --- | --- |
| `getElementById()` | Seleciona um único elemento pelo ID único. | `document.getElementById('logo')` |
| `getElementsByClassName()` | Retorna uma lista de elementos com aquela classe. | `document.getElementsByClassName('item')` |
| `getElementsByTagName()` | Retorna todos os elementos de uma tag específica. | `document.getElementsByTagName('p')` |
| `querySelector()` | Seleciona o **primeiro** elemento que combine com o seletor CSS. | `document.querySelector('.classe #id')` |
| `querySelectorAll()` | Seleciona **todos** os elementos que combinem com o seletor CSS. | `document.querySelectorAll('li.ativo')` |

---

### Exercício Prático

Considere o seguinte trecho HTML:

```html
<div id="container">
    <h1 class="titulo">Bem-vindo ao Curso</h1>
    <p>Lista de tarefas:</p>
    <ul class="lista">
        <li>Estudar HTML</li>
        <li>Praticar DOM</li>
    </ul>
</div>

```

**Desafios:**

1. **Seleção por ID:** Capture a `div` com o id "container" e exiba no console.
2. **Seleção por Classe:** Altere a cor do texto do elemento com a classe `titulo` para azul.
3. **Seleção Genérica:** Use `querySelectorAll` para contar quantos itens (`li`) existem na lista.
4. **Manipulação Simples:** Selecione o primeiro `li` e altere o seu conteúdo de texto para "Estudar JavaScript".

---

> **Dica:** Prefira utilizar `querySelector` e `querySelectorAll` para seleções mais complexas, pois eles utilizam a mesma sintaxe que você já aprendeu no CSS.


## 1. Mergulhando em Vetores (Arrays)

Um **Vetor** é uma variável especial que consegue armazenar uma lista ordenada de dados. Imagine que você é um professor e precisa guardar as notas de 40 alunos. Em vez de criar `nota1, nota2, nota3...`, você cria um único vetor `notas`.

### Anatomia de um Vetor

Todo vetor possui três características cruciais que você precisa dominar:

1. **Índice (A Chave):** É a posição do dado. Pense no índice como o número do armário em um vestiário. No JavaScript, o primeiro armário é sempre o **0**.
2. **Elemento (O Conteúdo):** É o que está guardado lá dentro (um número, um texto, ou até um elemento HTML).
3. **Comprimento (`length`):** É o total de espaços no vetor.

**Exemplo de Manipulação Manual:**

```javascript
let nomes = ["Ana", "Beto", "Caio"];

console.log(nomes[0]); // Saída: Ana
nomes[1] = "Beatriz";  // Altera "Beto" para "Beatriz"
console.log(nomes.length); // Saída: 3

```

---

## 2. Dinâmica: Vetores + DOM (`getElementsBy`)

A mágica acontece aqui: os métodos `getElementsByTagName` ou `getElementsByClassName` não devolvem apenas um valor, eles devolvem uma **coleção** (que funciona como um vetor).

Se você tem 10 parágrafos na tela e usa `document.getElementsByTagName('p')`, o JavaScript cria um vetor onde:

* Na posição `[0]` está o primeiro parágrafo.
* Na posição `[1]` está o segundo, e assim por diante.

### Por que usar o Loop `for`?

Como os elementos estão "presos" dentro de um vetor, você não consegue falar com todos de uma vez. Você precisa de um loop que faça o papel de um "entregador" que passa de porta em porta (índice por índice).

**A lógica do "Varredura":**

```javascript
// Capturamos todos os elementos com a classe 'caixa'
let caixas = document.getElementsByClassName('caixa');

// i começa em 0; enquanto i for menor que o total de caixas; aumente i
for (let i = 0; i < caixas.length; i++) {
    // Aplicamos a mudança individualmente em cada posição [i]
    caixas[i].style.backgroundColor = "blue";
    caixas[i].innerHTML = "Sou a caixa número " + i;
}

```

## 3. Exercício de Consolidação

Para testar se você entendeu a relação entre o **Índice do Vetor** e o **Elemento HTML**, realize este desafio:

**Cenário:** Você tem uma lista de compras (`<li>`) e quer destacar os itens.

1. **HTML:** Crie uma lista (`<ul>`) com pelo menos 5 itens (`<li>`).
2. **Captura:** No JavaScript, armazene todos os `<li>` em uma variável chamada `listaItens`.
3. **Processamento (O Loop):**
* Crie um loop `for` que percorra toda a extensão da `listaItens`.
* Dentro do loop, use uma estrutura condicional (`if`) para verificar:
* Se o **índice** for par, mude o fundo do item para **cinza claro**.
* Se o **índice** for ímpar, mude o fundo do item para **branco**.




4. **Personalização:** No final do loop, mude a fonte apenas do **último elemento** do vetor para negrito (Dica: o último elemento está na posição `length - 1`).

---

> **Reflexão Técnica:** Se o seu vetor tem 5 itens, o `length` é 5, mas os índices são 0, 1, 2, 3 e 4. Se você tentar acessar o índice 5, o JavaScript retornará `undefined`, pois você está tentando abrir um armário que não existe.
