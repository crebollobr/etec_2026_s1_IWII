# 🎓 Plano de Aula: Construindo uma Dashboard Inteligente

## 1. COOKIES (Persistência de Preferências)

### 📖 1. Explicação
Cookies permitem que o site se lembre do usuário sem exigir login. O evento `window.onload` é o nosso "porteiro": assim que o usuário entra, ele verifica as configurações salvas e ajusta o site automaticamente.

### 💻 2. Exemplo: Boas-vindas e Identificação
```html
<body onload="verificarAcesso()">
    <div id="box-login">
        <input type="text" id="user" placeholder="Seu apelido...">
        <button onclick="configurar()">Entrar no Sistema</button>
    </div>
    <h1 id="boas_vindas"></h1>

    <script>
        function configurar() {
            let nome = document.getElementById('user').value;
            // Salva o cookie com expiração de 1 hora
            document.cookie = "nickname=" + nome + "; path=/; max-age=3600";
            location.reload(); // Recarrega para aplicar o onload
        }

        function verificarAcesso() {
            if (document.cookie.includes("nickname=")) {
                let valor = document.cookie.split("nickname=")[1].split(";")[0];
                document.getElementById('boas_vindas').innerText = "Painel de: " + valor;
                document.getElementById('box-login').style.display = "none";
            }
        }
    </script>
</body>
```

### 🔬 3. Lab: Customização de Interface (O Site "Camaleão")
* **Tarefa:** Implemente um seletor de "Temas de Estudo". Crie três botões: "Foco" (fundo cinza escuro), "Criativo" (fundo roxo) e "Padrão" (fundo branco).
* **Requisito:** Ao clicar, salve o tema escolhido em um cookie. Ao carregar a página (`onload`), o site deve aplicar a cor de fundo e a cor da fonte correspondente.
* **Resultado Esperado:** Um site que "lembra" a identidade visual escolhida pelo aluno, mesmo que ele feche o navegador ou reinicie o computador.

## 2. XML (Feed de Notícias em Tempo Real)

### 📖 1. Explicação
O XML é o formato preferido para feeds de notícias (RSS). O JS recebe o texto, o `DOMParser` cria o objeto, e nós tratamos os itens como um **Vetor**. Através da propriedade `.href`, transformamos títulos estáticos em links reais de navegação.

### 💻 2. Exemplo: O "Card" de Notícia Principal
```html
<div class="card" style="border: 1px solid #ccc; padding: 15px; width: 300px;">
    <h3>Destaque do Dia</h3>
    <a id="link_xml" href="#" target="_blank" style="color: blue; text-decoration: none;">Carregando...</a>
    <p id="data_xml" style="font-size: 12px; color: #666;"></p>
</div>

<script>
    window.addEventListener('load', () => {
        fetch('https://curso.chr.eti.br/dados/noticias.xml')
            .then(res => res.text())
            .then(xmlTexto => {
                let xml = new DOMParser().parseFromString(xmlTexto, "text/xml");
                let noticias = xml.getElementsByTagName("item"); // VETOR DE ITENS
                
                // Pegamos a primeira notícia [0]
                let titulo = noticias[0].querySelector("title").textContent;
                let link = noticias[0].querySelector("link").textContent;
                let data = noticias[0].querySelector("pubDate").textContent;

                document.getElementById("link_xml").innerText = titulo;
                document.getElementById("link_xml").href = link;
                document.getElementById("data_xml").innerText = "Publicado em: " + data;
            });
    });
</script>
```

### 🔬 3. Lab: Mural de Notícias (Grid)
* **Tarefa:** De acordo com o exemplo, implemente um **Mural de Notícias** que exiba as 3 primeiras notícias do feed simultaneamente.
* **Requisito:** Crie três blocos (cards) no HTML. No Javascript, acesse as posições `[0]`, `[1]` e `[2]` do vetor de itens e preencha cada bloco com seu respectivo título, link e data.
* **Resultado Esperado:** Uma interface de portal de notícias onde o usuário pode ver o "Top 3" manchetes de tecnologia assim que a página abre.

## 3. JSON (Consumo de APIs Governamentais)

### 📖 1. Explicação
O JSON é a linguagem das APIs modernas. Ele já chega como um **Vetor de Objetos**. Usamos o `.forEach` para varrer a lista e o `document.createElement` para construir elementos de interface (como Selects ou Tabelas) sem precisar escrever o HTML manualmente.

### 💻 2. Exemplo: O Seletor de Estados Inteligente
```html
<label>Selecione seu Estado:</label>
<select id="select_estados" onchange="verInfo()">
    <option value="">Carregando...</option>
</select>
<p id="contador"></p>

<script>
    let dadosEstados = []; // Vetor global para guardar os dados

    fetch('https://curso.chr.eti.br/dados/estados.json')
        .then(res => res.json())
        .then(vetor => {
            dadosEstados = vetor; 
            let select = document.getElementById("select_estados");
            select.innerHTML = "<option>Escolha uma UF</option>";
            
            document.getElementById("contador").innerText = "Total: " + vetor.length + " estados.";

            vetor.forEach(estado => {
                let opt = document.createElement("option");
                opt.value = estado.sigla;
                opt.text = estado.nome;
                select.add(opt);
            });
        });

    function verInfo() {
        let sigla = document.getElementById("select_estados").value;
        alert("Você selecionou a sigla: " + sigla);
    }
</script>
```

### 🔬 3. Lab: Dashboard Geográfica
* **Tarefa:** Implemente um painel que, ao selecionar o estado, mostre o nome completo e a sigla em destaque em um quadro estilizado.
* **Requisito:** Utilize o evento `onchange` do Select. Quando o usuário mudar o estado, use o vetor para exibir: "Estado: [Nome] | Sigla: [Sigla]". Além disso, mostre quem é o "Primeiro" e o "Último" estado do vetor original nos cantos da tela.
* **Resultado Esperado:** Uma ferramenta de consulta onde a interface reage instantaneamente à escolha do usuário.

## 💀 DESAFIO INTEGRADO: O Localizador com Memória de Sessão

**O Projeto:** Criar um buscador de cidades que integra as 3 tecnologias.

1.  **Fase 1 (JSON/API):** O usuário seleciona o Estado no Select (criado no Lab 3). Ao selecionar, o JS dispara um `fetch` para a API real do IBGE: `https://servicodados.ibge.gov.br/api/v1/localidades/estados/{SIGLA}/municipios`.
2.  **Fase 2 (Vetor/DOM):** O resultado do IBGE é um vetor de cidades. Use o `.forEach` para listar todas as cidades em uma tabela estilizada.
3.  **Fase 3 (Cookies/Onload):** Ao realizar a busca, grave a sigla no cookie `ultima_uf`.
4.  **O "Tcham":** Ao dar F5, o site deve ler o cookie, selecionar o estado automaticamente no menu e **carregar a lista de cidades sem o usuário clicar em nada**.

**O que se espera ao final das 4h:** Um aluno que entende que o dado viaja (XML/JSON) e o navegador o mantém vivo (Cookies).
