# Parte I — Teoria

## 1. Por que sair dos atalhos

Até agora você usou três atalhos:

```javascript
$.get(url, callback);
$.getJSON(url, callback);
$.post(url, dados, callback);
```

Eles cobrem o caminho feliz. Mas não conseguem:

- Definir **timeout** (e se a rede demorar 30s, a sua página fica esperando pra sempre?).
- Reagir a **erro** de forma diferente do sucesso (e se o servidor responder 500?).
- Mandar **PUT** ou **DELETE** (esses dois métodos vamos usar na próxima aula).
- Adicionar **headers** customizados.
- Mandar o corpo em **JSON puro** com `Content-Type: application/json` (algumas APIs modernas exigem).

Pra tudo isso, existe a forma completa: **`$.ajax({...})`**.

## 2. A forma completa

```javascript
$.ajax({
    url: 'https://curso.chr.eti.br/ajax/api.php',
    method: 'GET',
    data: { aluno_id: 'a1b2c3d4e5f6g7h8' },
    dataType: 'json',
    timeout: 5000,
    success: function(resp) { ... },
    error: function(xhr, status, erro) { ... },
    complete: function() { ... }
});
```

Você passa **um único objeto** com todas as opções.

| Opção         | O que é                                                        |
|---------------|----------------------------------------------------------------|
| `url`         | endereço                                                       |
| `method`      | `'GET'`, `'POST'`, `'PUT'`, `'DELETE'` (default `'GET'`)        |
| `data`        | objeto → vira `?a=1&b=2` (GET) ou corpo (POST)                  |
| `dataType`    | o tipo da resposta esperada: `'json'`, `'html'`, `'text'`       |
| `contentType` | tipo do **corpo enviado** (`'application/json'` etc.)           |
| `timeout`     | ms antes de desistir                                            |
| `headers`     | objeto `{ 'X-Custom': 'valor' }`                                |
| `success`     | callback de sucesso                                             |
| `error`       | callback de erro                                                |
| `complete`    | callback que **sempre** roda, no fim                            |

**Ordem dos callbacks:** roda `success` **OU** `error` (um dos dois), depois `complete`. Sempre nessa ordem.

## 3. O objeto `xhr` no callback de erro

Quando dá erro, o callback `error` recebe três parâmetros:

```javascript
error: function(xhr, status, erro) {
    console.log(xhr.status);        // 404, 500, 0 (rede caiu)…
    console.log(xhr.statusText);    // "Not Found", "Internal Server Error"…
    console.log(xhr.responseText);  // o corpo do erro (geralmente HTML ou JSON)
    console.log(status);             // string: "error", "timeout", "abort", "parsererror"
    console.log(erro);               // mensagem (varia)
}
```

**O que cada `xhr.status` significa:**

| Status | Significado                                                       |
|--------|-------------------------------------------------------------------|
| 200    | OK (sucesso — não cai no `error`, cai no `success`)               |
| 201    | Created (POST que criou algo)                                     |
| 400    | Bad Request (você mandou algo inválido)                           |
| 401    | Unauthorized (precisa de login)                                   |
| 403    | Forbidden (logado, mas sem permissão)                             |
| 404    | Not Found (URL não existe)                                        |
| 500    | Internal Server Error (o servidor explodiu)                       |
| 0      | A requisição **nem saiu** ou **nem voltou** (rede caiu, DNS, CORS) |

## 4. `.done`, `.fail`, `.always` — outra forma de escrever a mesma coisa

`$.ajax` (e todos os atalhos) devolve um **objeto promise**. Em vez de passar `success/error/complete` dentro do objeto de configuração, dá pra **encadear**:

```javascript
$.ajax({ url: API, dataType: 'json' })
    .done(function(resp) {
        // sucesso (= success)
    })
    .fail(function(xhr) {
        // erro (= error)
    })
    .always(function() {
        // sempre (= complete)
    });
```

Funciona com os atalhos também:

```javascript
$.getJSON(url)
    .done(function(d) { ... })
    .fail(function(x) { ... });
```

**Por que isso é melhor que `success/error` dentro do objeto?**

1. **Leitura linear**, de cima pra baixo, sem aninhar.
2. Você pode adicionar mais `.done` ou `.fail` **em outros pontos do código** (ex: a função `criarTarefa` devolve a promise, e quem chama decide o que fazer com sucesso/erro):
   ```javascript
   function criarTarefa(titulo) {
       return $.post(API, { titulo: titulo });
   }

   // em outro lugar:
   criarTarefa('Comprar pão').done(carregarLista).fail(mostrarErro);
   ```
   Essa separação vai ser fundamental na Aula 5.

## 5. Várias requisições em paralelo

Quando você precisa de dados de **dois lugares ao mesmo tempo**, **não encadeie** (o segundo não depende do primeiro). Lance os dois **juntos**:

```javascript
$.getJSON('url-a').done(renderA).fail(mostraErroA);
$.getJSON('url-b').done(renderB).fail(mostraErroB);
```

As duas saem ao mesmo tempo, voltam quando voltarem. Você ganha tempo total (em vez de A→B sequencial).

**Cada uma tem `done`/`fail` independente.** Se a A falhar, ainda assim B funciona. Cada parte da UI atualiza sozinha.

**Quando você precisa esperar todas terminarem** pra fazer algo (ex: esconder o loading geral só quando todas voltaram), use **`$.when`**:

```javascript
$.when($.getJSON('a'), $.getJSON('b'), $.getJSON('c'))
    .done(function(respA, respB, respC) {
        // todas vieram com sucesso
        // CUIDADO: cada respX é um array [dados, status, xhr]
        var a = respA[0], b = respB[0], c = respC[0];
    })
    .fail(function() {
        // se QUALQUER UMA falhar, cai aqui
    });
```

Esse comportamento "falha total se uma falhar" é restritivo. Pra UI granular (cada widget falha sozinho), use `.done`/`.fail` em cada chamada independentemente. **Vamos usar a forma independente nos exercícios.**

## 6. `.load()` — componentizar HTML

`.load(url)` baixa **HTML** de uma URL e enfia direto dentro de um elemento. É a forma mais rápida de **componentizar** sua página — ter o mesmo header, menu, rodapé em várias páginas, sem repetir o HTML em cada uma.

```javascript
$('#topo').load('header.html');
$('#rodape').load('footer.html');
```

Quando o aluno editar `header.html`, **todas** as páginas que usam vão refletir a mudança no próximo F5.

**Você pode pegar só um pedaço** usando seletor:

```javascript
$('#topo').load('header.html #nav');
// baixa header.html INTEIRO, mas só pega o elemento #nav dele
```

**Atenção — restrição de protocolo:** `.load()` faz uma requisição HTTP. Se você abrir o HTML por **`file://...`**, o navegador bloqueia (CORS). Precisa abrir por **HTTP** — daí a importância de salvar tudo na sua pasta `curso.chr.eti.br/SEUNOME/`.

## 7. Estados de loading **por área**

Quando você tem **vários widgets** em uma página, cada um carregando dados de URLs diferentes, **cada um** tem seus próprios estados (carregando, sucesso, erro). Não tem "loading global" — tem **loading de cada widget**.

Isso é o que sites profissionais fazem. O Twitter, Facebook, Gmail — todos têm várias áreas que carregam em ritmos diferentes.

```
+-----------------+ +-----------------+ +-----------------+
| Suas tarefas    | | Novidades       | | Calendário      |
|                 | |                 | |                 |
| (carregando)    | | ✓ 5 itens       | | ✗ Falhou        |
|                 | |                 | |                 |
+-----------------+ +-----------------+ +-----------------+
```

A esquerda pode estar carregando, o do meio já chegou, e o da direita falhou. Cada um vive sua vida.

## 8. Pattern: "tentar de novo"

Erro de rede acontece. Boa UI: mostra o erro **e** oferece um botão "Tentar de novo".

```javascript
function carregar() {
    $('#tentar').hide();
    $('#estado').text('Carregando...').removeClass('err');

    $.ajax({ url: API, dataType: 'json', timeout: 5000 })
        .done(function(d) {
            // ok, renderiza
        })
        .fail(function() {
            $('#estado').text('Falhou. Veja sua conexão.').addClass('err');
            $('#tentar').show();
        });
}

$('#tentar').on('click', carregar);
carregar();
```

O usuário vê "Falhou" → aperta "Tentar de novo" → roda de novo. Sem F5.
