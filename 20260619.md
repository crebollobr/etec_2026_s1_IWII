# Parte teórica

## 1. Por que sair dos atalhos

`$.get`, `$.post`, `$.getJSON` são atalhos. Funcionam ótimo no caminho feliz. Mas você não tem como:

- Definir `timeout`.
- Reagir a erro de forma diferente do sucesso.
- Mandar `PUT` ou `DELETE` (esses dois métodos vamos usar na próxima aula).
- Adicionar headers customizados.

Pra tudo isso: **`$.ajax({...})`**.

## 2. A forma completa

```javascript
$.ajax({
    url: 'https://curso.chr.eti.br/ajax/api.php',
    method: 'GET',
    data: { aluno_id: 'ana123' },
    dataType: 'json',
    timeout: 5000,
    success: function(resp) { ... },
    error: function(xhr, status, erro) { ... },
    complete: function() { ... }
});
```

| Opção         | Pra que serve                                                  |
|---------------|----------------------------------------------------------------|
| `url`         | endereço                                                        |
| `method`      | GET, POST, PUT, DELETE                                          |
| `data`        | objeto que vira `?a=1&b=2` (GET) ou corpo (POST)                |
| `dataType`    | `'json'`, `'html'`, `'text'`                                    |
| `timeout`     | milissegundos antes de desistir                                 |
| `success`     | callback de sucesso                                             |
| `error`       | callback de erro                                                |
| `complete`    | callback que **sempre** roda (no fim)                           |

Ordem: roda `success` OU `error`, depois `complete`.

## 3. O objeto `xhr` no `error`

Quando dá erro, o callback recebe `(xhr, status, erro)`:

```javascript
xhr.status         // 404, 500, 0 (se rede caiu)…
xhr.statusText     // "Not Found"
xhr.responseText   // o que veio no corpo (texto)
xhr.responseJSON   // se veio JSON
status             // "error", "timeout", "abort", "parsererror"
```

## 4. `.done`, `.fail`, `.always`

`$.ajax` devolve um **objeto promise**. Em vez de passar `success/error` no objeto:

```javascript
$.ajax({ url: API, dataType: 'json' })
    .done(function(resp) { /* sucesso */ })
    .fail(function(xhr)  { /* erro */ })
    .always(function()   { /* sempre */ });
```

Funciona com `$.get`, `$.post`, `$.getJSON` também:

```javascript
$.getJSON(url)
    .done(function(d) { ... })
    .fail(function(x) { ... });
```

Vantagem: lê de cima pra baixo, sem aninhar.

## 5. Várias requisições em paralelo

Quando você precisa de dados de **dois lugares** ao mesmo tempo, **não encadeie** — lance os dois juntos:

```javascript
$.getJSON('url-a').done(renderA).fail(mostraErroA);
$.getJSON('url-b').done(renderB).fail(mostraErroB);
```

Cada um falha/sucede independente. Uma falha não derruba a outra.

## 6. `.load()` — componentizar

`.load(url)` baixa **HTML** de outra URL e enfia direto dentro de um elemento.

```javascript
$('#topo').load('header.html');
```

Útil pra ter o mesmo header/menu/rodapé em várias páginas. **Tem que rodar via HTTP** (não `file://`).

Você pode pegar **só um pedaço** usando seletor:

```javascript
$('#topo').load('header.html #nav');
```
