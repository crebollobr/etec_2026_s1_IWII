## Mudança importante: backend de verdade

Até agora a gente usou **JSONPlaceholder**. Você até podia fazer POST nela, mas ela **fingia** — devolvia status 201 e um objeto com id, sem salvar nada. Hoje trocamos por uma API **que salva mesmo**.

### A API do curso

URL: **`https://curso.chr.eti.br/ajax/api.php`**

Recurso único: **tarefas** (id, aluno_id, titulo, feito, criado_em).

| O que faz             | Como                                                       |
|-----------------------|------------------------------------------------------------|
| Listar suas tarefas   | `GET    api.php?aluno_id=ana123`                           |
| Pegar uma tarefa      | `GET    api.php?aluno_id=ana123&id=7`                      |
| Criar tarefa          | `POST   api.php` + corpo `{aluno_id, titulo}`              |
| Atualizar (aula 5)    | `PUT    api.php?id=7` + corpo `{aluno_id, titulo, feito}`  |
| Apagar (aula 5)       | `DELETE api.php?id=7&aluno_id=ana123`                      |

### `aluno_id` — o que é

A turma toda usa o mesmo servidor, então a API **filtra tudo** por `aluno_id`. **Cada um escolhe a sua identificação** e usa em todas as chamadas. Sem login, sem senha — não é segurança, é organização.

Sugestão: **`primeironome + 3 dígitos`** (ex: `ana123`, `bruno007`, `gustavo542`).

Hoje você digita seu `aluno_id` uma vez, ele vai pro `localStorage`, e fica salvo no navegador. Todas as chamadas AJAX hoje (e nas próximas 2 aulas) vão mandar esse `aluno_id` junto.

## 1. `localStorage`

### Teoria

`localStorage` é uma caixa de chave-valor **dentro do navegador** que persiste mesmo depois de fechar.

```javascript
localStorage.setItem('aluno_id', 'ana123');
var id = localStorage.getItem('aluno_id');   // 'ana123'
localStorage.removeItem('aluno_id');         // apaga
```

É **separado por site** (chr.eti.br não vê o que google.com guarda). Só guarda **string**.

## 2. POST com `$.post`

### Teoria

```javascript
$.post(url, dados, function(resposta) {
    // resposta = o que o servidor devolveu (geralmente JSON)
}, 'json');
```

- `dados` é um objeto JS. jQuery transforma em `chave=valor&chave=valor`.
- O último parâmetro `'json'` indica que esperamos resposta em JSON.

**Fluxo:**

```
[Página]                   [api.php]                  [SQLite]
   |--POST {aluno_id, titulo}->                          |
   |                          |--INSERT-------------->|
   |                          |<-id novo--------------|
   |<--{id, aluno_id, titulo, feito, criado_em}--------|
   |
[atualiza UI]
```

## 3. Formulários com `.serialize()`

### Teoria

Em vez de pegar campo por campo com `.val()`:

```javascript
var dados = { aluno_id: ALUNO_ID, titulo: $('#titulo').val(), prazo: $('#prazo').val() };
```

Você pode usar **`.serialize()`** num `<form>` inteiro — monta a string `name1=val1&name2=val2` automaticamente, usando o atributo `name` dos campos.

```html
<form id="f">
    <input name="titulo">
    <input name="prazo" type="date">
</form>
```

```javascript
$('#f').serialize();
// "titulo=Estudar+AJAX&prazo=2026-05-20"
```

**Importante:** `e.preventDefault()` segura o comportamento padrão do form (recarregar a página).

```javascript
$('#f').on('submit', function(e) {
    e.preventDefault();
    var dados = $(this).serialize();
    $.post(url, dados, callback, 'json');
});
```

### Demo guiado — `lab23/ex03-form-lista.html`

```html
<!DOCTYPE html>
<html lang="pt-BR">
<head>
    <meta charset="UTF-8">
    <title>Ex03 — Form + lista</title>
    <style>
        body { font-family: sans-serif; max-width: 700px; margin: 30px auto; }
        form { background: #f5f8ff; padding: 14px; border-radius: 6px; display: flex; gap: 8px; }
        form input { padding: 8px; font-size: 16px; flex: 1; }
        form button { padding: 8px 16px; font-size: 16px; }
        .tarefa { display: flex; gap: 10px; padding: 10px; border-bottom: 1px solid #eee; }
        .tarefa.feita { text-decoration: line-through; color: #888; }
        .vazio { color: #888; font-style: italic; padding: 20px 0; }
    </style>
</head>
<body>
    <h1>Minhas tarefas</h1>
    <p>Logado como: <strong id="quem"></strong></p>

    <form id="f">
        <input type="text" name="titulo" placeholder="Nova tarefa..." autofocus>
        <button type="submit">Adicionar</button>
    </form>

    <h2>Lista</h2>
    <div id="lista" class="vazio">Carregando...</div>

    <script src="https://code.jquery.com/jquery-3.7.1.min.js"></script>
    <script>
        var ALUNO_ID = localStorage.getItem('aluno_id');
        var API = 'https://curso.chr.eti.br/ajax/api.php';
        $('#quem').text(ALUNO_ID);

        function carregarTarefas() {
            $.getJSON(API + '?aluno_id=' + encodeURIComponent(ALUNO_ID), function(tarefas) {
                if (tarefas.length === 0) {
                    $('#lista').addClass('vazio').text('Nenhuma tarefa ainda.');
                    return;
                }

                $('#lista').removeClass('vazio').empty();

                $.each(tarefas, function(i, t) {
                    var $row = $('<div>').addClass('tarefa');
                    if (t.feito == 1) $row.addClass('feita');
                    $row.append($('<span>').text('#' + t.id + ' — ' + t.titulo));
                    $('#lista').append($row);
                });
            });
        }

        $('#f').on('submit', function(e) {
            e.preventDefault();
            var titulo = $('input[name="titulo"]', this).val().trim();
            if (!titulo) return;

            var dados = $(this).serialize() + '&aluno_id=' + encodeURIComponent(ALUNO_ID);

            $.post(API, dados, function() {
                $('input[name="titulo"]', '#f').val('').focus();
                carregarTarefas();
            }, 'json');
        });

        carregarTarefas();
    </script>
</body>
</html>
```

**Crie tarefas.** A lista atualiza sozinha. Feche, reabra: tudo lá. **Salvou de verdade.**

## 4. Validação visual antes de enviar

### Teoria

Quanto mais cedo você impedir um envio inútil, melhor.

```javascript
var titulo = $('#titulo').val().trim();

if (titulo.length < 3) {
    $('#campoTitulo').addClass('erro');
    $('#msgTitulo').text('Mínimo 3 caracteres.');
    return;
}
```

Padrão visual: pinte **a borda do campo** de vermelho e mostre a mensagem **perto** do campo.

**Bug clássico:** o aluno tenta enviar vazio, vê o erro, digita certo, e o erro **continua na tela**. Conserto: limpe o erro assim que ele voltar a digitar.

```javascript
$('#titulo').on('input', function() {
    $('#campoTitulo').removeClass('erro');
});
```

## 🏆 Desafio

Crie **`lab23/desafio.html`** — **agenda de contatos**.

A `api.php` só conhece o recurso "tarefas" — mas a gente **abusa do campo `titulo`** pra guardar tudo de um contato como uma string concatenada:

```
"Pedro Silva | pedro@email.com | (11) 99999-0000"
```

### Requisitos

1. Formulário com 3 campos: **nome**, **email**, **telefone**.
2. Validação:
   - nome ≥ 2 caracteres
   - email contém `@`
   - telefone ≥ 8 caracteres
3. Ao enviar, junta os 3 numa string `"nome | email | telefone"` e cria via `$.post`.
4. Lista abaixo mostra os contatos, **separando os campos de novo** (use `.split(' | ')`).
5. A lista atualiza sozinha depois de cada cadastro.

### Visual

```
+----------------------------------+
| Agenda                           |
+----------------------------------+
| Nome:    [_________________]     |
| Email:   [_________________]     |
| Telefone:[_________________]     |
| [ Adicionar contato ]            |
+----------------------------------+
| Contatos:                        |
|  ─ Pedro Silva                   |
|     pedro@email.com — (11) 9999 |
|  ─ Ana Costa                     |
|     ana@email.com — (21) 8888   |
+----------------------------------+
```
