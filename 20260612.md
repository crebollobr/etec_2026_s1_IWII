# Parte teórica

## 1. Backend de verdade

Até agora a gente usou **JSONPlaceholder**. Ela até aceita POST, mas **finge** — devolve status 201 e um id, sem salvar nada. Hoje trocamos por uma API **que salva mesmo**.

### A API do curso

URL: **`https://curso.chr.eti.br/ajax/api.php`**

Recurso único: **tarefas** (`id`, `aluno_id`, `titulo`, `feito`, `criado_em`).

| O que faz             | Como                                                       |
|-----------------------|------------------------------------------------------------|
| Listar suas tarefas   | `GET    api.php?aluno_id=ana123`                           |
| Pegar uma tarefa      | `GET    api.php?aluno_id=ana123&id=7`                      |
| Criar tarefa          | `POST   api.php` + corpo `{aluno_id, titulo}`              |
| Atualizar (aula 5)    | `PUT    api.php?id=7` + corpo JSON                         |
| Apagar (aula 5)       | `DELETE api.php?id=7&aluno_id=ana123`                      |

### O que é `aluno_id`

A turma toda usa o mesmo servidor, então a API **filtra tudo** por `aluno_id`. **Cada um escolhe a sua** identificação e usa em todas as chamadas. Sem login, sem senha — não é segurança, é organização.

Sugestão: **`primeironome + 3 dígitos`** (ex: `ana123`, `bruno007`).

## 2. `localStorage`

Caixa de chave-valor que persiste no navegador (mesmo depois de fechar). Vocês já viram em JS puro:

```javascript
localStorage.setItem('aluno_id', 'ana123');
var id = localStorage.getItem('aluno_id');   // 'ana123'
localStorage.removeItem('aluno_id');         // apaga
```

Só guarda **string**. É separado por site.

## 3. `$.post`

```javascript
$.post(url, dados, function(resposta) {
    // resposta = o que o servidor mandou de volta
}, 'json');
```

- `dados` é um objeto JS. jQuery transforma em `chave=valor&chave=valor` (form URL-encoded).
- O 4º parâmetro `'json'` diz pra esperar JSON na resposta.

## 4. `.serialize()` em formulários

Em vez de pegar campo por campo com `.val()`, num `<form>` inteiro:

```javascript
$('#meuForm').serialize()
// "titulo=Estudar+AJAX&prazo=2026-05-20"
```

Usa o atributo `name` de cada campo. Combinado com submit:

```javascript
$('#meuForm').on('submit', function(e) {
    e.preventDefault();          // não recarrega a página!
    var dados = $(this).serialize();
    $.post(url, dados, callback, 'json');
});
```

`e.preventDefault()` segura o comportamento padrão do form (recarregar).

## 5. Validação visual

Antes de enviar, conferir. Quando der erro:

- Pinte a **borda do campo** de vermelho.
- Mostre a mensagem **perto** do campo.
- **Limpe o erro** quando o aluno voltar a digitar:

```javascript
$('#titulo').on('input', function() {
    $('#campoTitulo').removeClass('erro');
});
```
