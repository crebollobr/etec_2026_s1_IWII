

# Sextou com AJAX: Dominando Requisições no Front-end

Nesta aula de sexta-feira à noite, vamos mergulhar fundo no processamento de scripts no lado do cliente utilizando Javascript. O grande astro da noite são as solicitações assíncronas, tradicionalmente conhecidas como AJAX. 

Prepare o café e vamos destrinchar como a web moderna busca dados por trás dos panos!

## 1. O que é AJAX, afinal?

**AJAX** significa *Asynchronous JavaScript and XML* (JavaScript e XML Assíncronos). 

No passado, a web era "síncrona" e travada. Se você preenchesse um formulário e clicasse em "Enviar", a página inteira ficava branca, o navegador pensava, e uma página totalmente nova era carregada. 

O AJAX mudou o jogo. Ele permite que o navegador converse com o servidor em segundo plano (de forma assíncrona), peça dados e atualize apenas um pedacinho da tela, sem recarregar a página inteira.
* **A Analogia do Restaurante:** Sem AJAX, você faz o pedido no balcão e fica travado lá esperando a comida ficar pronta. Com AJAX, você faz o pedido, senta na mesa, continua conversando (usando a página), e o garçom traz a comida apenas quando estiver pronta.

> **A História do "X":** Originalmente, os dados viajavam no formato XML. Hoje em dia, a esmagadora maioria dos sistemas usa o formato **JSON** (JavaScript Object Notation), que é muito mais leve e nativo do Javascript. A sigla deveria ser AJAJ, mas AJAX já tinha pegado!

**Exemplos do AJAX no mundo real:**
* **Rolagem Infinita:** Quando você desce o feed do Instagram e novos posts aparecem magicamente.
* **Auto-completar:** Quando você digita no Google e as sugestões aparecem embaixo antes de você dar Enter.
* **Curtir uma foto:** O coração fica vermelho instantaneamente enquanto o AJAX avisa o servidor silenciosamente.



## 2. Os Comandos (Métodos) HTTP

Quando o AJAX vai até o servidor buscar ou levar uma informação, ele precisa dizer **o que** ele quer fazer. Ele faz isso usando os métodos HTTP (os "verbos" da internet). 

| Método | O que faz? | Exemplo de uso prático |
| :--- | :--- | :--- |
| **GET** | Buscar / Ler dados. | Pedir os dados de um CEP ou a lista de usuários. (É o padrão do navegador). |
| **POST** | Criar / Enviar dados novos. | Enviar os dados de um formulário de cadastro de um novo cliente. |
| **PUT / PATCH**| Atualizar dados existentes. | Alterar a sua foto de perfil ou mudar a sua senha. |
| **DELETE** | Apagar dados. | Clicar no ícone de lixeira para excluir um e-mail. |

## 3. Dissecando o Código: A Busca de Usuários

Abaixo temos um código HTML e Javascript completo e funcional. Ele pega um ID digitado, vai até um servidor na internet, busca os dados do usuário correspondente e preenche os campos na tela sem recarregar a página.

**O Código Base:**
```html
<html>
<body>
<table>
<tr><td>id</td><td><input type="text" id="id"></td></tr>
<tr><td>Consulta</td><td><input type="button" onclick="pesquisaUsuario()" value="Pesquisar"></td></tr>
<tr><td>name</td><td><input type="text" id="name"></td></tr>
<tr><td>email</td><td><input type="text" id="email"></td></tr>
<tr><td>username</td><td><input type="text" id="username"></td></tr>
<tr><td>company_name</td><td><input type="text" id="company_name"></td></tr>
</table>
</body>
<script>
const id = document.getElementById('id');
const name = document.getElementById('name');
const email = document.getElementById('email');
const username = document.getElementById('username');
const company_name = document.getElementById('company_name');

function limpa_formulario() {
    id.value = "";
    name.value = "";
    email.value = "";
    username.value = "";
    company_name.value = "";
}

function pesquisaUsuario() {
    var id_valor = id.value;

    if (id_valor != "") {

        name.value = "...";
        email.value = "...";
        username.value = "...";
        company_name.value = "...";

        fetch(`https://jsonplaceholder.typicode.com/users/${id_valor}`)
            .then(resposta => resposta.json())
            .then(dados => {
                if (dados.id) {
                     name.value = dados.name;
                     email.value = dados.email;
                     username.value = dados.username;
                     company_name.value = dados.company.name;
                } else {
                    limpa_formulario();
                    alert("Usuário não encontrado.");
                }
            })
            .catch(() => {
                limpa_formulario();
                alert("Erro na requisição.");
            });

    } else {
        limpa_formulario();
    }
}
</script>
</html>
```

### Explicação Linha por Linha do Javascript:

1.  **`const id = document.getElementById('id');` (e as linhas seguintes):** O Javascript cria "ganchos". Ele vai até o HTML e captura os campos de texto pelo `id` deles. Agora, o JS tem controle sobre essas caixinhas na tela.
2.  **`function limpa_formulario() { ... }`**: Uma função auxiliar simples que esvazia todos os campos colocando texto vazio `""` neles.
3.  **`function pesquisaUsuario() { ... }`**: A função principal que é disparada quando o botão "Pesquisar" é clicado.
4.  **`var id_valor = id.value;`**: Captura exatamente o número que o usuário digitou dentro da caixinha de ID.
5.  **`if (id_valor != "") { ... }`**: Verifica se o usuário não deixou o campo em branco antes de tentar ir para a internet.
6.  **`name.value = "...";` (e seguintes):** Preenche os campos com reticências. Isso é crucial para a experiência do usuário. Como a internet tem atraso, isso avisa: *"Calma, estou indo buscar"*.
7.  **`fetch(https://jsonplaceholder.../${id_valor})`**: **O coração do AJAX.** É o comando moderno do Javascript para fazer a requisição HTTP. Ele diz: *"Vá até essa URL (injetando o ID no final) e traga os dados"*. O código *não* fica travado aqui esperando.
8.  **`.then(resposta => resposta.json())`**: A palavra `then` significa "então" (uma *Promise*). Ele diz: *"Quando o servidor responder, **então** pegue o pacote bruto e converta para o formato JSON"*.
9.  **`.then(dados => { ... })`**: Uma vez convertido, entramos no segundo `.then`. Agora temos um objeto Javascript real chamado `dados`.
10. **`if (dados.id) { ... }`**: Verifica se o retorno do servidor realmente contém um usuário válido (se a propriedade `id` existe no JSON).
11. **`name.value = dados.name;` (e seguintes):** A mágica acontece! Ele pega a informação da rede (`dados.name`) e injeta no HTML (`name.value`).
12. **`.catch(() => { ... })`**: E se a internet cair? É para isso que serve o `catch`. Ele captura falhas de rede (quando o servidor nem consegue ser alcançado) e evita que a aplicação quebre silenciosamente, exibindo um alerta.

## 4. O Laboratório: Desafio do CEP

Agora é com vocês! O desafio do laboratório é criar uma página de busca de CEP que funcione com a mesma simplicidade do código de busca de usuários que acabamos de estudar. 

**Vocês vão usar o código de Usuários como modelo base.** A estrutura do HTML e a lógica do `fetch`, dos `.then()` e do `.catch()` serão praticamente idênticas.

No entanto, para construir essa solução, vocês precisam agir como investigadores de dados.

### A Fonte da Informação
Vocês vão consumir os dados do **ViaCEP**, um serviço público e gratuito. 
A URL (endpoint) que vocês vão colocar dentro da função `fetch()` do Javascript segue este formato:
`https://viacep.com.br/ws/SEU_CEP_AQUI/json/`

### O Formato dos Dados (A Investigação)
Antes de programar qualquer linha nova, abram o navegador de internet de vocês, colem a URL `https://viacep.com.br/ws/13042105/json/` na barra de endereços e deem Enter.

O servidor vai devolver os dados do endereço em formato **JSON**. O resultado será exatamente este:

```json
{
  "cep": "13042-105",
  "logradouro": "Avenida Washington Luís",
  "complemento": "de 1001/1002 ao fim",
  "unidade": "",
  "bairro": "Vila Marieta",
  "localidade": "Campinas",
  "uf": "SP",
  "estado": "São Paulo",
  "regiao": "Sudeste",
  "ibge": "3509502",
  "gia": "2446",
  "ddd": "19",
  "siafi": "6291"
}
```

### Entendendo os Campos
Para que o Javascript de vocês consiga pegar a informação correta e jogar no formulário HTML, vocês precisam usar os nomes exatos das chaves (as palavras que estão à esquerda dos dois pontos no JSON acima).

Repare bem nos detalhes dos dados fornecidos pela API:
* **`logradouro`**: É o campo que guarda o nome da rua ou avenida. No sistema do ViaCEP, a palavra "rua" não existe, você deve extrair o dado usando `dados.logradouro` (ou `conteudo.logradouro`, dependendo de como você nomear a variável no `.then`).
* **`bairro`**: Guarda o nome do bairro.
* **`localidade`**: É a chave responsável por armazenar o nome da cidade. Atenção a esse nome!
* **`uf`**: Guarda a sigla do estado (Unidade Federativa).
* **`ibge`**: Traz o código numérico do município.

2.  Crie a função `pesquisacep()`.
3.  Adapte o Javascript para usar a URL do ViaCEP e "casar" as caixinhas de texto do HTML com as chaves corretas que vocês investigaram no JSON.
