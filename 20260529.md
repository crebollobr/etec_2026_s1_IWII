# Parte I — Teoria

## 1. Por que listas mudam o jogo

Na Aula 1 você pediu **um** post e mostrou ele. Agora pense numa rede social: ela mostra dezenas de posts de uma vez. Numa loja virtual: dezenas de produtos. Numa lista de contatos: centenas de pessoas.

Você **não vai escrever** um `<div>` no HTML pra cada item. Você vai pedir um **array** de dados ao servidor, percorrer esse array e **gerar HTML para cada item** na hora. Isso é **renderização dinâmica**.

Isso é o que React, Vue e Angular fazem o tempo todo. A diferença é que em jQuery você faz "na mão" — o que é ótimo pra entender o mecanismo.

## 2. JSON e arrays vindos da API

JSON é o formato padrão de troca de dados na web. Parece muito com objeto JavaScript:

```json
{
    "id": 7,
    "titulo": "Estudar AJAX",
    "feito": false
}
```

Quando a API devolve uma **lista** de coisas, vem um **array de objetos**:

```json
[
    { "id": 1, "title": "post um",   "body": "..." },
    { "id": 2, "title": "post dois", "body": "..." }
]
```

Por exemplo, esta URL devolve 10 usuários (cada um com nome, email, endereço, empresa):

```
https://jsonplaceholder.typicode.com/users
```

## 3. `$.getJSON` — o atalho pra JSON

`$.get` funciona com qualquer resposta (texto, HTML, JSON). Mas quando você **sabe** que vem JSON, use o atalho `$.getJSON`:

```javascript
$.getJSON(url, function(dados) {
    // 'dados' já está parseado como objeto/array JS
});
```

Diferença prática: nenhuma, na vida cotidiana. `$.get` também detecta JSON automaticamente. Mas `$.getJSON` deixa **explícito** no código que você espera JSON — fica mais legível.

## 4. Iterando com `$.each`

Pra percorrer um array (ou objeto), o jQuery oferece o utilitário **`$.each`**:

```javascript
var frutas = ['maçã', 'pera', 'uva'];

$.each(frutas, function(i, fruta) {
    console.log(i, fruta);
});
// 0 maçã
// 1 pera
// 2 uva
```

- `i` é o **índice** (0, 1, 2...)
- `fruta` é o item atual

Vale também para objetos:

```javascript
$.each({nome: 'Ana', idade: 30}, function(chave, valor) {
    console.log(chave, valor);
});
// nome Ana
// idade 30
```

Você poderia usar `for` ou `forEach` do JavaScript puro — `$.each` é só uma conveniência do jQuery, com a vantagem de funcionar igual com arrays e objetos.

**Não confunda** com `.each` (sem `$`), que é diferente: este percorre uma **coleção jQuery** (elementos do DOM), enquanto `$.each` percorre array/objeto JavaScript:

```javascript
$.each([1, 2, 3], function(i, n) { ... });     // percorre array
$('.item').each(function(i) { ... });          // percorre elementos DOM
```

Os dois existem e fazem coisas parecidas, em contextos diferentes.

## 5. Criando elementos: `$('<tag>')` vs concatenação de string

Tem duas maneiras de adicionar HTML novo ao DOM. Vamos comparar.

**Maneira A — concatenando string:**

```javascript
$('#mural').append(
    '<div class="post">' +
        '<h3>' + p.title + '</h3>' +
        '<p>' + p.body + '</p>' +
    '</div>'
);
```

Funciona, mas tem dois problemas:

1. **Insegurança.** Se `p.title` vier do servidor com `<script>alert(1)</script>`, esse script vai executar na sua página. (No JSONPlaceholder isso não acontece, mas em sistemas reais é vulnerabilidade séria — chama-se **XSS**.)
2. **Difícil de ler** quando o card é grande.

**Maneira B — criando elemento por elemento com `$('<tag>')`:**

```javascript
var $bloco = $('<div>').addClass('post');
$bloco.append($('<h3>').text(p.title));
$bloco.append($('<p>').text(p.body));
$('#mural').append($bloco);
```

- `$('<div>')` cria um `<div>` novo (na memória, não na tela ainda).
- `.text(p.title)` **escapa** o conteúdo — se vier `<script>`, aparece como texto literal, não executa.
- `.append($bloco)` no fim coloca tudo na tela.

**Convenção:** quando uma variável guarda um objeto jQuery, comece o nome com `$` (`$bloco`, `$linha`). Lembra que ali tem métodos jQuery.

**Quando usar cada um?** Pra dados que **vêm do servidor** ou **do usuário**, use a Maneira B (`.text()`). Pra HTML totalmente sob seu controle (literal, sem variável), a Maneira A é OK.

## 6. Estados de UI: a regra dos 3 (ou 4) estados

Toda interface que carrega dados do servidor passa por estados:

| Estado          | Quando                                   | O que mostrar                            |
|-----------------|------------------------------------------|------------------------------------------|
| **Carregando**  | requisição em andamento                  | "Carregando..." em cinza, ou um spinner  |
| **Vazio**       | resposta veio mas array está vazio       | "Nenhum resultado." em cinza             |
| **Sucesso**     | resposta veio com dados                  | os dados em si                           |
| **Erro**        | requisição falhou                        | "Erro." em vermelho (veremos na Aula 4)  |

A tela **nunca pode ficar em branco** enquanto carrega. Se ficar, o usuário pensa que está quebrado e fecha. Sempre dê **algum sinal** de que algo está acontecendo.

**Truque útil pra testar o loading** quando sua rede é rápida demais e o "Carregando..." some em 50ms:

```javascript
setTimeout(function() {
    $.getJSON(url, callback);
}, 1500);   // atrasa 1.5s antes de chamar
```

Aí você consegue ver o loading direito.

## 7. Busca client-side

Quando os dados **já estão na tela**, dá pra filtrar **sem nova requisição** — só esconde o que não bate.

Padrão típico:

```javascript
$('#busca').on('input', function() {
    var termo = $(this).val().toLowerCase();
    $('.item').each(function() {
        var texto = $(this).text().toLowerCase();
        $(this).toggle(texto.indexOf(termo) !== -1);
    });
});
```

Decomposto:

- `.on('input', ...)` — dispara a **cada tecla** digitada no campo (diferente de `.on('change', ...)`, que só dispara quando o campo perde foco).
- `$(this).val().toLowerCase()` — pega o que está digitado, em minúsculas (pra comparação case-insensitive).
- `$('.item').each(...)` — percorre cada elemento `.item` no DOM.
- Dentro do `each`, `$(this)` é o item atual.
- `texto.indexOf(termo) !== -1` — `indexOf` devolve a posição onde encontrou, ou **-1** se não encontrou. Então `!== -1` significa "encontrou em algum lugar".
- `.toggle(boolean)` — se `true`, mostra; se `false`, esconde.

**Filtro `:visible`** — pra contar só os que estão visíveis após filtrar:

```javascript
$('.item:visible').length   // quantos passaram pelo filtro
```

## 8. Acessando propriedades aninhadas

JSON real tem propriedades dentro de propriedades. Por exemplo, o objeto do usuário da JSONPlaceholder:

```json
{
    "name": "Clementine Bauch",
    "address": {
        "city": "McKenziehaven",
        "zipcode": "59590-4157"
    },
    "company": {
        "name": "Romaguera-Jacobson"
    }
}
```

Você acessa com **ponto** ou **colchete**:

```javascript
user.name                  // "Clementine Bauch"
user.address.city          // "McKenziehaven"
user.company.name          // "Romaguera-Jacobson"
user['address']['city']    // mesma coisa, sintaxe alternativa
```

Acesse o que precisar do objeto. **Não tem como adivinhar a estrutura** — sempre dê um `console.log(user)` na primeira vez que receber para conhecer.

## 9. Query strings — parâmetros na URL

Muitas APIs aceitam **parâmetros** na URL, depois de `?`, separados por `&`:

```
https://jsonplaceholder.typicode.com/posts?userId=1&_limit=10
```

Isso quer dizer: "me dá os posts do userId=1, no máximo 10". É padronizado.

No jQuery, em vez de montar a string na mão (sujeita a erros), você pode usar a opção `data`:

```javascript
$.getJSON('https://jsonplaceholder.typicode.com/posts', {
    userId: 1,
    _limit: 10
}, function(posts) { ... });
```

Vai montar a URL `?userId=1&_limit=10` automaticamente.

## 10. Encadeando duas (ou três) requisições

Às vezes você precisa de **um post** e depois **o autor desse post**. Você só sabe o `userId` depois que o post chega — então a segunda chamada **depende** da primeira:

```javascript
$.getJSON('https://jsonplaceholder.typicode.com/posts/1', function(post) {
    // post chegou. Agora pede o autor:
    $.getJSON('https://jsonplaceholder.typicode.com/users/' + post.userId, function(autor) {
        // post E autor disponíveis
        $('#tela').html('<h2>' + post.title + '</h2><small>por ' + autor.name + '</small>');
    });
});
```

Funciona, mas tem um problema chamado **callback hell**: quando você precisa de 3, 4, 5 requisições em sequência, fica isto:

```javascript
$.get('...', function(a) {
    $.get('...', function(b) {
        $.get('...', function(c) {
            $.get('...', function(d) {
                // ... ilegível
            });
        });
    });
});
```

Pelas próximas duas aulas, ainda vamos viver com isso. **Na Aula 4** vamos aprender a versão mais limpa com `.done()`.
