# Parte I — Teoria

## 1. Por que jQuery em 2026?

jQuery é uma biblioteca JavaScript lançada em **2006**, num momento em que cada navegador implementava JavaScript de um jeito diferente. Antes do jQuery, fazer uma chamada AJAX que funcionasse no Internet Explorer, no Firefox e no Safari ao mesmo tempo dava **25 linhas de código**, cheias de `if (navegador X) ... else ...`. Com jQuery, viraram 3 linhas que funcionavam em todos.

A frase de campanha foi: *"write less, do more"*. Funcionou: em 2010, **mais da metade dos sites do mundo** usava jQuery.

Hoje, 2026, o cenário mudou. Os navegadores convergiram, o JavaScript moderno tem `fetch()`, e frameworks como React, Vue e Angular dominam a construção de aplicações grandes. Apesar disso, **~70% dos sites do mundo ainda usam jQuery** — WordPress, Bootstrap, intranets corporativas, sistemas administrativos antigos.

Então por que **você** vai aprender jQuery agora? Por três razões:

1. **Manutenção.** Cedo ou tarde você vai pegar um sistema legado pra mexer. Saber jQuery é diferencial.
2. **Didática.** jQuery é o jeito mais limpo e curto de aprender o que é AJAX, o que é manipulação de DOM, o que é callback. Os conceitos são os mesmos que React/Vue usam por baixo — só ficam mais visíveis aqui.
3. **Produtividade.** Pra protótipos rápidos e páginas pequenas com algum interativo, jQuery resolve em 5 linhas o que em vanilla JS leva 20.

> **AJAX em uma frase:** pedir dados ao servidor em segundo plano e atualizar **só uma parte** da página, sem recarregar tudo.
>
> AJAX não é uma linguagem, não é um framework. É uma **técnica**, que apareceu por volta de 2005 quando Gmail e Google Maps mostraram pro mundo que dava pra web parecer aplicativo de desktop.

## 2. O DOM — como o JavaScript enxerga a página

Antes do jQuery, precisamos entender uma ideia central. Quando o navegador carrega um HTML, ele não guarda aquele texto como uma string. Ele constrói **uma árvore** chamada **DOM** (*Document Object Model*).

Imagine este HTML:

```html
<body>
    <h1 id="titulo">Mercado</h1>
    <ul>
        <li>Arroz</li>
        <li>Feijão</li>
    </ul>
</body>
```

Internamente, vira mais ou menos isto:

```
body
├── h1 #titulo  ───── "Mercado"
└── ul
    ├── li ───── "Arroz"
    └── li ───── "Feijão"
```

Cada caixa nessa árvore é um **objeto**. O JavaScript pode pegar referência pra qualquer um deles e mexer — mudar o texto, mudar a cor, adicionar filhos, remover. **É isso que jQuery facilita.**

JavaScript puro pra pegar o `<h1>`:

```javascript
var t = document.getElementById('titulo');
t.textContent = 'Feira';
t.style.color = 'crimson';
```

Mesmo trabalho em jQuery:

```javascript
$('#titulo').text('Feira').css('color', 'crimson');
```

Mais curto, mais legível, e funciona igual em qualquer navegador. O jQuery é, na essência, **uma camada elegante por cima do DOM**.

## 3. A função `$()` — a porta de entrada do jQuery

`$` é só um **apelido** para a função `jQuery`. As duas linhas abaixo fazem exatamente a mesma coisa:

```javascript
$('#titulo').text('Olá');
jQuery('#titulo').text('Olá');
```

Usamos `$` porque é mais curto. Mas saiba: `$` **não é mágica do JavaScript**, é só um identificador que a biblioteca jQuery define quando carrega.

A função `$()` aceita uma **string com seletor CSS** e devolve uma **coleção** de elementos. Por exemplo:

```javascript
$('p')        // coleção com TODOS os <p> da página
$('#menu')    // coleção com o (único) elemento de id "menu"
$('.aviso')   // coleção com todos os elementos de classe "aviso"
```

**Importante:** mesmo quando o seletor casa com **um único** elemento, o jQuery devolve uma coleção (de tamanho 1). Você pode conferir o tamanho:

```javascript
$('p').length         // ex: 4
$('#menu').length     // 1
$('.naoExiste').length // 0  (não dá erro — só uma coleção vazia)
```

A grande sacada da coleção é: **quando você chama um método nela, o método age em TODOS os elementos**. Sem `for`:

```javascript
$('p').css('color', 'blue');   // pinta todos os <p> de azul de uma vez
```

Esse é o superpoder do jQuery. Você não itera — descreve **o conjunto** e age sobre ele.

### Carregando o jQuery numa página

jQuery é um único arquivo `.js`. Carregar é simples:

```html
<script src="https://code.jquery.com/jquery-3.7.1.min.js"></script>
```

Coloque essa linha **antes do `</body>`**. Se colocar no `<head>`, seu script tenta mexer em elementos que ainda nem foram desenhados — bug silencioso. Colocando no final do body, quando o seu script roda, tudo já está na tela.

Confira que carregou abrindo o console (F12):

```javascript
$.fn.jquery   // "3.7.1"
```

Se aparecer `$ is not defined`, o caminho do `<script>` está errado ou o jQuery não carregou.

## 4. Seletores

A regra simples é: **toda sintaxe CSS que funciona no navegador, funciona em `$()`**. Além disso, o jQuery adiciona **filtros próprios**. Vamos por partes.

### 4.1 Por tag

```javascript
$('p')      // todos os <p>
$('h2')     // todos os <h2>
$('img')    // todas as <img>
```

Sem `#`, sem `.`. Só o nome da tag.

### 4.2 Por id

`id` é um atributo único — cada id deve aparecer **uma única vez** numa página.

```html
<h1 id="titulo">...</h1>
```

```javascript
$('#titulo')   // o elemento com id="titulo"
```

O `#` antes do nome é obrigatório. Sem ele, jQuery procuraria uma tag chamada `titulo`.

### 4.3 Por classe

Classe é um rótulo que você pode aplicar a **vários** elementos. Diferente de id, classe pode repetir:

```html
<p class="aviso">Importante!</p>
<p>Comum.</p>
<p class="aviso">Outro aviso.</p>
```

```javascript
$('.aviso')   // os 2 parágrafos de classe "aviso"
```

O **ponto** antes do nome é obrigatório.

**Múltiplas classes num mesmo elemento.** Um elemento pode ter várias classes, separadas por **espaço** no atributo `class`:

```html
<li class="prato vegetariano caro">Risoto trufado</li>
<li class="prato">Bife</li>
<li class="prato vegetariano">Salada</li>
```

Esse primeiro `<li>` tem **três** classes: `prato`, `vegetariano` e `caro`. Você pode selecioná-lo por qualquer uma:

```javascript
$('.prato')         // todos os 3 <li>
$('.vegetariano')   // só os 2 vegetarianos
$('.caro')          // só o trufado
```

Para exigir que tenha **as duas classes ao mesmo tempo**, escreva sem espaço:

```javascript
$('.prato.vegetariano')   // só os <li> que têm AS DUAS classes
```

Se você colocar espaço entre as duas, vira outra coisa (veja seletores hierárquicos abaixo).

### 4.4 União com vírgula

```javascript
$('h1, h2, h3')   // todos os <h1>, todos os <h2> e todos os <h3>
```

Vírgula = OU. Pega tudo que casar com qualquer um dos seletores.

### 4.5 Hierárquicos

Quando você quer pegar elementos que estão dentro de outros.

| Seletor    | O que significa                                          |
|------------|----------------------------------------------------------|
| `ul li`    | qualquer `<li>` **descendente** de algum `<ul>` (em qualquer nível de profundidade) |
| `ul > li`  | só os `<li>` **filhos diretos** de algum `<ul>`          |
| `h1 + p`   | o `<p>` **imediatamente depois** de um `<h1>` (irmão adjacente) |
| `h1 ~ p`   | qualquer `<p>` **irmão depois** de um `<h1>`             |

Exemplo. Dado este HTML:

```html
<ul>
    <li>A</li>
    <li>B
        <ul>
            <li>B1</li>
            <li>B2</li>
        </ul>
    </li>
</ul>
```

- `$('ul li')` pega 4 elementos (A, B, B1, B2).
- `$('ul > li')` pega 2 (A e B), mas só os que são filhos diretos do `<ul>` de fora. B1 e B2 não entram porque o pai direto deles é o `<ul>` interno (que também é pego pelo seletor — então também conta).

Ou seja: `ul > li` casa com qualquer `<li>` cujo pai direto seja `<ul>`. No exemplo, isso inclui A e B do externo, e B1 e B2 do interno. Total: 4. Quem fica de fora? Nenhum, neste caso.

Agora compare com este HTML diferente:

```html
<div>
    <ul>
        <li>A</li>
    </ul>
    <span><li>fora</li></span>
</div>
```

- `$('ul li')` pega só A — porque o segundo `<li>` não está dentro de nenhum `<ul>`.
- `$('ul > li')` pega só A — pelo mesmo motivo.

### 4.6 Por atributo

Você pode selecionar pelos atributos que o elemento tem:

| Sintaxe                  | Pega                                              |
|--------------------------|---------------------------------------------------|
| `[atributo]`             | que tem o atributo (com qualquer valor)           |
| `[atributo="x"]`         | atributo igual a `x`                              |
| `[atributo^="x"]`        | atributo **começa com** `x`                       |
| `[atributo$="x"]`        | atributo **termina com** `x`                      |
| `[atributo*="x"]`        | atributo **contém** `x`                           |

Exemplos práticos:

```javascript
$('input[type="text"]')      // só inputs de texto (não os de senha, número etc)
$('a[href^="https"]')        // links que começam com https (externos)
$('img[src$=".png"]')        // imagens PNG
$('a[href*="etec"]')         // links cujo href contém "etec"
```

### 4.7 Filtros do jQuery (não são CSS)

Esses são exclusivos do jQuery — começam com `:` e ficam **depois** do seletor base.

| Filtro              | Pega                                            |
|---------------------|-------------------------------------------------|
| `:first`            | o primeiro elemento da coleção                  |
| `:last`             | o último                                        |
| `:even`             | os de índice par (0, 2, 4...)                   |
| `:odd`              | os de índice ímpar (1, 3, 5...)                 |
| `:eq(n)`            | o elemento de índice `n`                        |
| `:contains("x")`    | os que contêm o texto "x" (em qualquer lugar dentro) |
| `:has(seletor)`     | os que **têm** o seletor dentro                 |
| `:not(seletor)`     | os que **não** batem com o seletor              |

Exemplos:

```javascript
$('li:first')              // primeira <li> da página
$('tr:odd')                // linhas ímpares de tabela (zebra striping!)
$('p:contains("AJAX")')    // parágrafos que mencionam AJAX
$('div:has(img)')          // divs que têm pelo menos uma imagem dentro
$('input:not([disabled])') // inputs que não estão desabilitados
```

Atenção a uma armadilha: **os índices começam em 0**. Então `:eq(0)` pega o primeiro, `:eq(2)` pega o terceiro. E `:odd` (ímpar) pega os de índice 1, 3, 5... que visualmente são a **2ª, 4ª, 6ª** linhas. Faz mais sentido visualmente do que numericamente.

## 5. Mexer no DOM

Selecionar é metade do trabalho. A outra metade é **mexer** no que você selecionou.

### 5.1 Texto: `.text()` vs `.html()`

```javascript
$('#titulo').text('Olá <b>mundo</b>');
// → o texto literal "Olá <b>mundo</b>" aparece (com os <b> visíveis)

$('#titulo').html('Olá <b>mundo</b>');
// → o <b> é interpretado como HTML, "mundo" fica em negrito
```

Use `.text()` quando o conteúdo é texto puro. Use `.html()` quando precisa de tags. **Quando o dado vem de fora (servidor, formulário), prefira `.text()`** — é mais seguro, não interpreta `<script>` malicioso.

Para **ler** o conteúdo, é o mesmo método sem argumento:

```javascript
var atual = $('#titulo').text();   // pega o texto atual
```

### 5.2 Valor de formulário: `.val()`

`.val()` é exclusivo para **campos de formulário** — `<input>`, `<textarea>`, `<select>`. Não funciona em `<h1>`, `<p>`, `<div>`.

```javascript
$('#nome').val()              // lê o valor digitado
$('#nome').val('Ana')         // escreve um valor no campo
```

**Confusão clássica:** usar `.val()` em um `<h1>` (não funciona — use `.text()`). Use `.val()` **só** em campo de formulário.

### 5.3 Estilo inline: `.css()`

Aplica CSS direto no atributo `style` do elemento:

```javascript
$('#titulo').css('color', 'red');           // um par chave-valor
$('#titulo').css({                          // objeto com vários
    color: 'red',
    fontSize: '40px',
    background: 'yellow'
});
```

Nomes de propriedade com hífen no CSS (como `font-size`, `background-color`) viram **camelCase** no JS (`fontSize`, `backgroundColor`). Alternativa: usar a string com hífen entre aspas: `'font-size'`.

Para **ler** uma propriedade computada:

```javascript
$('#titulo').css('color');   // ex: "rgb(255, 0, 0)"
```

### 5.4 Classes CSS: `.addClass()`, `.removeClass()`, `.toggleClass()`

Em vez de pintar com `.css()` toda vez, é melhor **definir uma classe no CSS** e ligar/desligar a classe via JS:

```css
.destaque { background: yellow; color: red; font-weight: bold; }
```

```javascript
$('#item').addClass('destaque');     // aplica a classe
$('#item').removeClass('destaque');  // tira
$('#item').toggleClass('destaque');  // se tem, tira; se não tem, aplica
```

Vantagem: você muda toda a aparência num lugar só (no CSS), e o JS só decide **quando** aplicar. Código mais limpo e fácil de mudar depois.

Para perguntar se um elemento tem certa classe:

```javascript
if ($('#item').hasClass('destaque')) { ... }
```

### 5.5 Inserir elementos: `.append()`, `.prepend()`, `.after()`, `.before()`

Veja a diferença entre os 4 — fundamental:

```html
<div id="caixa">
    <p>conteúdo</p>
</div>
```

```javascript
$('#caixa').append('<p>novo</p>');    // insere DENTRO, no FIM
$('#caixa').prepend('<p>novo</p>');   // insere DENTRO, no INÍCIO
$('#caixa').after('<p>novo</p>');     // insere DEPOIS (irmão, fora)
$('#caixa').before('<p>novo</p>');    // insere ANTES (irmão, fora)
```

Resultado de cada um, partindo do HTML acima:

```html
<!-- append (dentro, no fim) -->
<div id="caixa">
    <p>conteúdo</p>
    <p>novo</p>
</div>

<!-- prepend (dentro, no início) -->
<div id="caixa">
    <p>novo</p>
    <p>conteúdo</p>
</div>

<!-- after (depois, irmão) -->
<div id="caixa">
    <p>conteúdo</p>
</div>
<p>novo</p>

<!-- before (antes, irmão) -->
<p>novo</p>
<div id="caixa">
    <p>conteúdo</p>
</div>
```

Você passa **uma string com HTML** ou **um objeto jQuery** (que veremos no Ex 6).

### 5.6 Remover: `.remove()` e `.empty()`

```javascript
$('#item').remove();   // apaga o próprio elemento (e tudo dentro)
$('#item').empty();    // apaga o CONTEÚDO mas mantém o elemento (vira <div id="item"></div>)
```

### 5.7 Efeitos visuais

```javascript
$('#caixa').hide();        // some imediatamente (display: none)
$('#caixa').show();        // reaparece

$('#caixa').fadeOut(800);  // some com fade em 800ms
$('#caixa').fadeIn(800);   // aparece com fade

$('#caixa').slideUp(800);   // recolhe pra cima
$('#caixa').slideDown(800); // desce
```

Úteis pra dar feedback visual ao usuário ("isto vai ser apagado" → fadeOut → remove).

## 6. Encadeamento (chaining)

A maioria dos métodos jQuery **devolve a própria coleção**. Isso permite encadear chamadas:

```javascript
$('#titulo')
    .text('Olá')
    .css('color', 'red')
    .addClass('grande')
    .fadeIn(500);
```

Lê-se assim: pegue o `#titulo`, mude o texto, pinte de vermelho, adicione a classe `grande`, e faça aparecer com fade. **Tudo na mesma coleção**, sem precisar repetir o `$('#titulo')`.

Métodos que **não** devolvem a coleção: aqueles que devolvem valor (como `.text()` sem argumento, `.css('color')` sem 2º argumento, `.val()` sem argumento, `.length`). Esses interrompem o encadeamento.

## 7. Eventos — recap rápido em jQuery

Você já viu `addEventListener` em JS puro. Em jQuery a sintaxe equivalente é:

```javascript
$('#botao').on('click', function() {
    // o que fazer quando clicar
});
```

Dentro da função, `$(this)` se refere ao elemento clicado (em jQuery), e `this` (sem `$`) é o elemento DOM puro. Quase sempre você quer `$(this)`.

Outros eventos comuns: `'input'` (toda tecla num campo), `'change'` (quando muda valor de select/checkbox), `'submit'` (formulário), `'keypress'` (tecla pressionada).

Para disparar um evento via código (sem o usuário clicar):

```javascript
$('#botao').trigger('click');   // simula clique
```

Útil pra rodar uma ação assim que a página abrir, por exemplo.

## 8. AJAX — o conceito

Antes do AJAX, qualquer interação com servidor recarregava a página **inteira**. Em 2004, Gmail mostrou que dava pra apagar um email e a lista atualizar **sem piscar**. Em 2005, Jesse James Garrett batizou a técnica: **A**synchronous **J**avaScript **a**nd **X**ML.

A ideia central: o JavaScript faz uma requisição ao servidor **em segundo plano** (o usuário continua usando a página), e quando a resposta volta, você atualiza só **uma parte** do DOM.

```
SEM AJAX:
[clique] → tela branca → página inteira nova

COM AJAX:
[clique] → JS pede dado (página segue viva) → atualiza só o pedaço necessário
```

Em jQuery, o jeito mais simples de fazer uma requisição GET é `$.get`:

```javascript
$.get('https://exemplo.com/dados', function(resposta) {
    // este código roda DEPOIS, quando a resposta chega
    console.log(resposta);
});
```

O 2º argumento é uma **função** — o **callback**. O jQuery guarda essa função e chama ela quando o servidor responder. Por isso "assíncrono": você não fica esperando a resposta na linha do `$.get`; o código continua.

```javascript
console.log('1');
$.get('https://...', function(resp) {
    console.log('3 — resposta chegou');
});
console.log('2');
```

No console aparece: **1, 2, 3**. A linha 3 imprime **depois** da 2, porque a 2 já estava na fila para rodar enquanto a rede ainda buscava os dados.

> **Regra ouro do AJAX:** se você precisa do dado, use **dentro do callback**. Tentar usar fora vai dar `undefined`, porque o código continua sem esperar a resposta.

### A API de teste — JSONPlaceholder

Vamos usar uma API pública gratuita chamada **JSONPlaceholder**, que devolve dados falsos pra estudo:

| URL                                                  | Devolve              |
|------------------------------------------------------|----------------------|
| `https://jsonplaceholder.typicode.com/posts/1`       | um post específico   |
| `https://jsonplaceholder.typicode.com/posts`         | lista de 100 posts   |
| `https://jsonplaceholder.typicode.com/users/3`       | usuário 3            |
| `https://jsonplaceholder.typicode.com/users`         | lista de 10 usuários |

Quando você faz `$.get` numa URL dessas, o jQuery percebe que a resposta é JSON e **já converte em objeto JavaScript** automaticamente. Então no callback você recebe um objeto direto:

```javascript
$.get('https://jsonplaceholder.typicode.com/posts/1', function(post) {
    // post é { id: 1, userId: 1, title: "...", body: "..." }
    console.log(post.title);
});
```

Isso fecha a parte teórica. **Tudo o que vem nos exercícios usa só o que está acima.**

   - Lê o texto do `#nova`, faz `.trim()`.
   - Se vazio, retorna (não adiciona nada).
   - Cria um `<li>` contendo:
     - Um `<span>` com o texto da tarefa.
     - Um `<button>` com texto "×".
   - Adiciona o `<li>` ao `#lista`.
   - Limpa o `#nova` e dá `.focus()`.
   - Chama `atualizarVazio()`.

   ```javascript
   $('#nova').on('keypress', function(e) {
       if (e.which === 13) adicionar();   // 13 = código do Enter

   ```

     $('#lista').on('click', 'button', function() {

     });
     ```
   A opção B é mais robusta. Use ela.

   });
   ```
