
# Aula: A Memória da Web - Cookies, Sessões e Segurança

## 1. A História: O "Post-it" de Lou Montulli
Em 1994, a web era um lugar solitário e sem memória. Cada vez que você clicava em um link, o servidor te tratava como um completo estranho. O problema? **Carrinhos de compra.** Sem memória, você colocava um item no carrinho, clicava em "Próximo" e o carrinho estava vazio de novo.

**Lou Montulli**, um engenheiro da Netscape, resolveu isso criando o **Cookie**. O nome vem de "magic cookies", um termo antigo da computação. A ideia era simples: o site dá ao seu navegador um pequeno pedaço de texto e diz: *"Guarde isso e me mostre toda vez que você voltar aqui"*.

## 2. Por que usar Cookies? A "Amnésia" do HTTP
O protocolo HTTP é **stateless** (sem estado). Ele não guarda histórico.
* **Sem Cookie:** O servidor é como um atendente de caixa com amnésia total que esquece quem você é assim que você vira as costas.
* **Com Cookie:** O servidor te dá um crachá. Na próxima vez, ele olha o crachá e sabe quem você é.

### Sessão: OSI vs. TCP/IP
* **Modelo OSI:** Possui uma camada inteira dedicada a isso (**Camada 5 - Sessão**). Ela deveria cuidar da abertura e fechamento da conversa.
* **Modelo TCP/IP (O que usamos na prática):** Não tem uma camada de sessão separada. A "sessão" acontece na **Camada de Aplicação**.
* **A Solução:** Usamos o Cookie para carregar o **Session ID** (Identificador de Sessão), simulando essa persistência que o protocolo não tem nativamente.

## 3. Segurança do Cookie: O Lado Obscuro
Nem tudo são flores. Como o cookie é um arquivo de texto no computador do usuário, ele pode ser roubado ou manipulado. Por isso, existem "travas" de segurança:

1.  **HttpOnly:** Se você marcar um cookie como `HttpOnly`, o Javascript do programador (ou de um hacker) **não consegue ler** o cookie. Isso evita ataques de XSS (Cross-Site Scripting).
2.  **Secure:** O cookie só é enviado se a conexão for **HTTPS** (criptografada).
3.  **SameSite:** Impede que o cookie seja enviado em requisições feitas por outros sites, protegendo contra ataques de CSRF (falsificação de requisição).

## 4. Manipulando Cookies

### No Backend (Exemplo em Node.js)
Diferente do Javascript do navegador, o servidor (Node.js) envia o cookie através do cabeçalho da resposta HTTP (`Set-Cookie`).

```javascript
// Exemplo simples usando o framework Express no Node.js
app.get('/login', (req, res) => {
    // Define um cookie chamado 'usuario' com valor 'Marcos'
    // que expira em 1 hora e é protegido (HttpOnly)
    res.cookie('usuario', 'Marcos', { 
        maxAge: 3600000, 
        httpOnly: true, 
        secure: true 
    });
    res.send('Cookie enviado com sucesso!');
});
```

### No Frontend (Exemplo Completo em Javascript)
No navegador, o `document.cookie` funciona de um jeito estranho: ele parece uma string, mas se comporta como um banco de dados.

```html
<script>
    // Função para CRIAR um cookie
    function criarCookie(nome, valor, dias) {
        const data = new Date();
        data.setTime(data.getTime() + (dias * 24 * 60 * 60 * 1000));
        let expires = "expires=" + data.toUTCString();
        // Grava o cookie no formato: nome=valor; expiração; caminho
        document.cookie = nome + "=" + valor + ";" + expires + ";path=/";
    }

    // Função para LER um cookie
    function lerCookie(nome) {
        let busca = nome + "=";
        let cookies = document.cookie.split(';');
        for(let i = 0; i < cookies.length; i++) {
            let c = cookies[i].trim();
            if (c.indexOf(busca) == 0) return c.substring(busca.length, c.length);
        }
        return "";
    }
</script>
```

## 5. Estudo de Caso: Experiência do Usuário (UX) - Modo Escuro
A melhor aplicação prática de cookies para UX é o **Modo Escuro (Dark Mode)**. Queremos que, se o usuário escolher o modo escuro, o site já carregue preto na próxima vez.

### O código do caso:
Para evitar aquele "flash branco" antes da página carregar, a verificação do cookie deve ocorrer no **`onload`** do `<body>` ou, melhor ainda, no início do script.

```html
<html>
<head>
    <style>
        body { background-color: white; color: black; transition: 0.5s; }
        .dark-mode { background-color: #222; color: white; }
        .painel { border: 1px solid #ccc; padding: 20px; margin-top: 20px; }
    </style>
</head>
<body onload="verificarTema()">

    <div class="painel">
        <h1>Meu Site Profissional</h1>
        <p>Escolha sua preferência de visual abaixo:</p>
        <button onclick="alternarTema()">Alternar Modo Escuro</button>
    </div>

    <script>
        function alternarTema() {
            let corpo = document.body;
            corpo.classList.toggle("dark-mode");

            // Salva a preferência no Cookie
            if (corpo.classList.contains("dark-mode")) {
                document.cookie = "tema=escuro; path=/";
            } else {
                document.cookie = "tema=claro; path=/";
            }
        }

        function verificarTema() {
            // Esta função roda assim que o BODY carrega
            let cookies = document.cookie;
            if (cookies.includes("tema=escuro")) {
                document.body.classList.add("dark-mode");
            }
        }
    </script>
</body>
</html>
```

## 7. Desafio extra03: O Tamagotchi de Navegador
**A Missão:** Crie um bichinho virtual (emoji ou imagem).
* Crie botões para "Alimentar" e "Brincar".
* Os níveis de **Fome** e **Felicidade** devem ser salvos em Cookies.
* **O objetivo:** O aluno deve fechar o navegador, abrir de novo e o bichinho deve continuar com o mesmo status de antes.

```
let campoSalva = lerCookie("campo");
let campo = (campoSalva !== "") ? parseInt(campoSalva) : 0;
// se o campoSalva não for vazio, usar o valor inteiro dele, se não usar o padrão 0
```

```
<html>
<head>
    <meta charset="utf-8">
    <title>Meu Bichinho Virtual</title>
    <style>
        body { text-align: center; font-family: sans-serif; padding: 50px; background-color: #f0f8ff; }
        #bichinho { font-size: 100px; margin: 20px; transition: 0.3s; }
        .status { font-size: 24px; margin: 10px; font-weight: bold; }
        .controles button { font-size: 18px; padding: 10px 20px; margin: 5px; cursor: pointer; border-radius: 8px; border: 1px solid #ccc; }
        .controles button:hover { background-color: #e0e0e0; }
    </style>
</head>
<body>

    <h1>Meu Tamagotchi</h1>
    <div id="bichinho">🐶</div>
    
    <div class="status">Fome: <span id="fome">0</span>%</div>
    <div class="status">Felicidade: <span id="felicidade">100</span>%</div>
    
    <div class="controles">
        <button onclick="alimentar()">🍎 Alimentar</button>
        <button onclick="brincar()">⚽ Brincar</button>
    </div>

    <script>
        // 1. As Variáveis de Estado (A memória de curto prazo) - implementar o cookie aqui
        let fome = 0;
        let felicidade = 100;

        // 2. Capturando os elementos da tela
        const textoFome = document.getElementById('fome');
        const textoFelicidade = document.getElementById('felicidade');
        const visualBichinho = document.getElementById('bichinho');

        // 3. Função para atualizar o visual
        function atualizarTela() {
            // Travas de segurança para não passar de 100% nem cair de 0%
            if (fome < 0) fome = 0;
            if (fome > 100) fome = 100;
            if (felicidade < 0) felicidade = 0;
            if (felicidade > 100) felicidade = 100;

            // Atualiza os números na tela
            textoFome.innerText = fome;
            textoFelicidade.innerText = felicidade;

            // Muda a carinha do bichinho dependendo do estado
            if (fome >= 80 || felicidade <= 20) {
                visualBichinho.innerText = "😵"; // Quase morrendo!
            } else if (fome >= 50 || felicidade <= 50) {
                visualBichinho.innerText = "😟"; // Triste ou com fome
            } else {
                visualBichinho.innerText = "🐶"; // Feliz e saudável!
            }
        }

        // 4. Ações do Usuário (Cliques)
        function alimentar() {
            fome -= 15; // Comer diminui a fome
            atualizarTela();
        }

        function brincar() {
            felicidade += 15; // Brincar aumenta a felicidade
            fome += 5; // Mas brincar gasta energia e dá fome!
            atualizarTela();
        }

        // 5. O CORAÇÃO DO JOGO: O Loop de Tempo (setInterval)
        // Esse bloco vai rodar sozinho a cada 2000 milissegundos (2 segundos)
        setInterval(() => {
            fome += 2; // Fome vai aumentando sozinha
            felicidade -= 1; // Felicidade vai caindo se ficar ignorado
            
            atualizarTela(); // Atualiza a tela a cada "batida" do coração
        }, 2000); 
        
    </script>
</body>
</html>
```
