# Parte I — Teoria

## 1. O ciclo CRUD

CRUD é a sigla das 4 operações básicas que **qualquer** sistema que gerencia dados precisa oferecer:

| Sigla     | Significa                  | Método HTTP | Onde vimos      |
|-----------|----------------------------|-------------|-----------------|
| **C**     | **C**reate (criar)         | POST        | Aula 3          |
| **R**     | **R**ead (ler/listar)      | GET         | Aulas 1, 2, 3   |
| **U**     | **U**pdate (atualizar)     | PUT         | **Hoje**        |
| **D**     | **D**elete (apagar)        | DELETE      | **Hoje**        |

Quase todo sistema do mundo é, no fundo, um CRUD com regras de negócio em cima. Cadastro de clientes? CRUD. Carrinho de compras? CRUD de itens. Posts numa rede social? CRUD. Tarefas? CRUD.

Hoje fechamos esse ciclo.

## 2. Por que PUT e DELETE não têm atalho

O jQuery oferece atalhos pros métodos mais comuns: `$.get`, `$.post`, `$.getJSON`. Mas **não** oferece `$.put` nem `$.delete`. Por dois motivos históricos:

1. **PUT e DELETE são menos usados no dia a dia** que GET e POST. Quando você navega em qualquer site, está fazendo mais GET que tudo junto.
2. **HTML não suporta PUT/DELETE nativamente** em formulário. Um `<form method="...">` aceita só GET e POST. Isso vem da especificação original do HTML.

Então pra PUT e DELETE, você usa a forma completa: **`$.ajax({ method: 'PUT', ... })`** ou **`$.ajax({ method: 'DELETE', ... })`**.

## 3. PUT — atualizar

PUT significa "**substituir** o recurso de id X pelos dados que estou enviando". Em prática, você usa pra:

- Mudar o título de uma tarefa.
- Marcar uma tarefa como concluída.
- Editar o nome de um contato.
- Trocar a senha de um usuário.

```javascript
$.ajax({
    url: 'https://curso.chr.eti.br/ajax/api.php?id=7',
    method: 'PUT',
    data: JSON.stringify({ aluno_id: 'ana123', titulo: 'Comprar leite', feito: 1 }),
    contentType: 'application/json',
    dataType: 'json'
})
.done(function(tarefa) {
    // 'tarefa' = versão atualizada que o servidor devolveu
});
```

**Diferenças cruciais em relação ao POST:**

1. O **id** vai na **query string** (`?id=7`), não no corpo. Faz sentido: o id identifica **qual** recurso atualizar; o corpo diz **com o que** atualizar.
2. O corpo é **JSON puro** (não form-urlencoded como o POST simples). Por isso:
   - `JSON.stringify({...})` — converte o objeto JS pra string JSON.
   - `contentType: 'application/json'` — diz ao servidor que o corpo é JSON.
3. **Atualização parcial** funciona — a `api.php` aceita só os campos que você mandar:

```javascript
// só marca como feito, sem mexer no título
data: JSON.stringify({ aluno_id: ALUNO_ID, feito: 1 })
```

Os campos que você **não** enviou permanecem como estavam.

## 4. DELETE — apagar

```javascript
$.ajax({
    url: API + '?id=7&aluno_id=' + ALUNO_ID,
    method: 'DELETE',
    dataType: 'json'
})
.done(function(resp) {
    // resp = { ok: true, id: 7 }
});
```

Características:

- **Sem corpo.** O id e o aluno_id vão na query string.
- A resposta geralmente é um JSON pequeno confirmando o que foi apagado.

**Regra de UX importante:** apagar é **destrutivo e irreversível**. **Sempre confirme** antes:

```javascript
if (!confirm('Apagar essa tarefa?')) return;
```

Ou implemente um padrão melhor (toast com "Desfazer", que vamos fazer num exercício). **Nunca apague sem perguntar**.

## 5. `JSON.stringify` e `JSON.parse`

Como você vai usar muito hoje, vale repassar:

```javascript
var obj = { id: 7, nome: 'Ana' };

var s = JSON.stringify(obj);   // '{"id":7,"nome":"Ana"}'  — string

var voltou = JSON.parse(s);     // { id: 7, nome: 'Ana' }   — objeto de novo
```

- `stringify` é usado pra **mandar** dados (transformar JS → string pra rede).
- `parse` é usado pra **receber** dados (string → JS). O jQuery faz isso automaticamente quando você usa `dataType: 'json'`.

## 6. `$.extend` — juntar dois objetos

Útil em CRUD: você tem `{ titulo: 'novo' }` (o que veio do form) e precisa adicionar o `aluno_id` antes de enviar.

```javascript
var dados = $.extend({ aluno_id: ALUNO_ID }, { titulo: 'novo' });
// dados = { aluno_id: 'ana123', titulo: 'novo' }
```

Útil pra construir uma função que aceita campos parciais:

```javascript
function atualizar(id, campos) {
    var body = $.extend({ aluno_id: ALUNO_ID }, campos);
    return $.ajax({
        url: API + '?id=' + id,
        method: 'PUT',
        data: JSON.stringify(body),
        contentType: 'application/json',
        dataType: 'json'
    });
}

atualizar(7, { feito: 1 });            // só marca feito
atualizar(7, { titulo: 'novo' });      // só renomeia
atualizar(7, { titulo: 'x', feito: 0 }); // ambos
```

## 7. Várias operações em sequência — esperando todas terminarem

Quando você quer apagar **várias** tarefas (botão "Limpar concluídas"), são **N requisições DELETE**. Você precisa esperar **todas** voltarem antes de recarregar a lista.

A forma elegante: junte as promises num array, e use `$.when.apply` pra esperar todas:

```javascript
var promises = [];
$.each(idsParaApagar, function(i, id) {
    promises.push(apagarTarefa(id));
});

$.when.apply($, promises).done(function() {
    // todas voltaram com sucesso
    carregarLista();
});
```

`$.when.apply($, promises)` é um jeito de chamar `$.when(promises[0], promises[1], ...)` quando você não sabe quantas são. É um truque clássico do JavaScript.

## 8. Organizando o código em 3 camadas

Até agora, em quase todo exercício, o JavaScript ficou empilhado dentro do `<script>` — funções de API, de UI e de evento misturadas. Funciona pra arquivos pequenos. Mas pro projeto final (50+ linhas de JS), vira caos.

A organização que **todo** desenvolvedor experiente segue:

```javascript
// === 1. CONFIG ===
var API = '...';
var ALUNO_ID = localStorage.getItem('aluno_id');
var filtroAtual = 'todas';

// === 2. CAMADA API ===
// Só fala com servidor. Cada função DEVOLVE uma promise.
function listarTarefas() {
    return $.getJSON(API + '?aluno_id=' + ALUNO_ID);
}
function criarTarefa(titulo) {
    return $.post(API, { aluno_id: ALUNO_ID, titulo: titulo }, null, 'json');
}
function atualizarTarefa(id, dados) {
    return $.ajax({
        url: API + '?id=' + id, method: 'PUT',
        data: JSON.stringify($.extend({ aluno_id: ALUNO_ID }, dados)),
        contentType: 'application/json', dataType: 'json'
    });
}
function apagarTarefa(id) {
    return $.ajax({
        url: API + '?id=' + id + '&aluno_id=' + ALUNO_ID,
        method: 'DELETE', dataType: 'json'
    });
}

// === 3. CAMADA UI ===
// Só desenha. Não faz chamadas AJAX.
function renderizar(tarefas) { ... }
function atualizarContador(tarefas) { ... }
function mostrarErro(xhr) { ... }

// === 4. CAMADA LIGAÇÃO (eventos) ===
// Conecta UI com API. Onde os eventos disparam.
function carregar() {
    listarTarefas().done(renderizar).fail(mostrarErro);
}

$('#formNovo').on('submit', function(e) {
    e.preventDefault();
    var titulo = $('#novoTitulo').val().trim();
    if (!titulo) return;
    criarTarefa(titulo).done(function() {
        $('#novoTitulo').val('').focus();
        carregar();
    });
});

// === 5. BOOT ===
if (ALUNO_ID) carregar();
```

**3 regras de ouro:**

1. **API não fala de UI** (não chama `$('#x')...`). Só servidor.
2. **UI não fala de servidor** (não chama `$.ajax`). Só DOM.
3. **Eventos são a ponte** — chamam API, no `.done` chamam UI.

Quando dá bug, você sabe **onde** olhar: bug de dado é na camada API; bug visual é na UI; bug de "não acontece nada quando clica" é em LIGAÇÃO.

É o mesmo princípio que React/Vue/Angular usam, só com mais ferramentas em volta.

## 9. Edição inline

Padrão moderno pra editar um item sem ir pra outra tela:

- Aluno clica (ou duplo-clica) no texto.
- O `<span>` vira um `<input>` com o texto atual selecionado.
- Aluno digita o novo texto.
- ENTER salva (PUT + recarrega).
- ESC cancela (sem PUT).
- Clicar fora do input também salva.

Conceitualmente:

```javascript
$titulo.on('dblclick', function() {
    var atual = $(this).text();
    var $input = $('<input type="text">').val(atual);

    $(this).replaceWith($input);
    $input.focus().select();   // seleciona todo o texto

    $input.on('blur', salvar);
    $input.on('keypress', function(e) {
        if (e.which === 13) salvar();   // Enter
    });
    $input.on('keydown', function(e) {
        if (e.which === 27) cancelar();  // Esc
    });
});
```

É um padrão de UI mais avançado que vale conhecer.

     - dataType: `'json'`
   - `.done`: `setMsg('ok', '✓ Salvo: ' + t.titulo)`.
   - `.fail`: `setMsg('err', 'Erro ao salvar (status: ' + xhr.status + ')')`.

```javascript
$('#lista').on('click', '.del', function() {
    var $row = $(this).closest('.tarefa');
    var id = $row.data('id');
    var titulo = $row.find('.titulo').text();

    if (!confirm('Apagar "' + titulo + '"?')) return;

    $row.addClass('removendo');

    apagar(id)
        .done(function() {
            $row.slideUp(300, function() { $(this).remove(); });
        })
        .fail(function(xhr) {
            $row.removeClass('removendo');
            alert('Erro ao apagar: ' + xhr.status);
        });
});
```

4. Chama `carregar()` ao abrir.

```html
<body>
    <h1>Lista (com desfazer)</h1>
    <div id="lista">Carregando...</div>

    <div id="toast">
        <span id="toast-msg"></span>
        <button id="desfazer">Desfazer</button>
    </div>

    <script src="https://code.jquery.com/jquery-3.7.1.min.js"></script>
    <script>
        // SEU CÓDIGO AQUI
    </script>

</html>
```

**O que fazer:**

1. Mesmas funções `listar`, `apagar`, `carregar`, `renderizar` do ex03.

2. **Variáveis globais pra controlar o desfazer**:
   ```javascript
   var timerDesfazer = null;
   var rowEscondida = null;   // a linha que sumiu (DOM)
   var idEscondido = null;
   ```

3. **Clique no `.del`** (delegation):
   - Pega a linha e o id.
   - **NÃO** chama `confirm`. Em vez disso:
     - Esconde a linha imediatamente: `$row.slideUp(200)`.
     - Salva referência em `rowEscondida = $row` e `idEscondido = id`.
     - Mostra o `#toast` com "Tarefa removida.".
     - Agenda o DELETE pra 5 segundos depois com `setTimeout`:
       ```javascript
       timerDesfazer = setTimeout(function() {
           apagar(idEscondido).always(function() {
               rowEscondida.remove();
               $('#toast').hide();
               rowEscondida = null;
               idEscondido = null;
           });
       }, 5000);
       ```

4. **Clique no `#desfazer`**:
   - `clearTimeout(timerDesfazer)` — cancela o DELETE agendado.
   - `rowEscondida.slideDown(200)` — a linha volta.
   - Esconde o `#toast`.
   - Zera as variáveis.

5. **Atenção**: se o usuário apagar uma segunda tarefa enquanto o toast da primeira ainda está visível, você precisa:
   - Antes de agendar o novo, **cancelar** o timer anterior e **executar** o DELETE pendente. (Ou seja, "selar" a primeira ação porque uma nova começou.)
   - Pode fazer assim: no início do clique no `.del`, se `timerDesfazer` não for null, dispare o DELETE pendente imediatamente antes de prosseguir.

6. **Teste**: apague uma → vê toast → clica desfazer → volta. Apague outra → espera 5s → some pra valer.

   - Atualiza `#msg` para "Apagando N..." em azul.
   - Para cada concluída, chama `apagar(id)` e guarda no array de promises.
   - `$.when.apply($, promises).done(function() { ... })`:
     - `#msg` vira "✓ N tarefas apagadas." em verde.
     - Chama `carregar()`.

4. **Botão "Recarregar"**: chama `carregar()`.

## Exercício 6 — Refatorando em 3 camadas

Copie seu **`lab25/ex03.html`** (apagador) para **`lab25/ex06.html`** e **reorganize** o JavaScript em 3 camadas, com comentários separadores e ordem fixa.

**O que fazer:**

1. No topo do script, defina **CONFIG**:
   ```javascript
   var API = 'https://curso.chr.eti.br/ajax/api.php';
   var ALUNO_ID = localStorage.getItem('aluno_id');
   ```

2. Em seguida, defina a **CAMADA API**. As funções **devem devolver a promise** do `$.ajax`/`$.getJSON`/`$.post`. **Não** chamam `.done` ou `.fail` aqui — quem chama é que decide.

   ```javascript
   // === API ===
   function listarTarefas() { ... }
   function apagarTarefa(id) { ... }
   ```

3. Depois, **CAMADA UI**. Funções que só desenham. Não fazem AJAX.

   ```javascript
   // === UI ===
   function renderizar(tarefas) { ... }
   function mostrarErro(xhr) { ... }
   ```

4. Por fim, **CAMADA LIGAÇÃO**. Onde os eventos disparam, chamando API + UI:

   ```javascript
   // === LIGAÇÃO ===
   function carregar() {
       listarTarefas()
           .done(renderizar)
           .fail(mostrarErro);
   }

   $('#lista').on('click', '.del', function() { ... });
   ```

5. E o **BOOT** no fim:
   ```javascript
   carregar();
   ```

## Pra continuar estudando depois do curso

Você fechou o ciclo completo de uma aplicação web cliente-servidor. Tudo que vier daqui pra frente é variação:

- **`fetch()` nativo** — versão moderna do `$.ajax`, sem precisar de jQuery. Em projetos novos, é o padrão.
- **`async / await`** — sintaxe nova pra escrever AJAX **sem callbacks** nem `.done().fail()`:
  ```javascript
  async function carregar() {
      var resp = await fetch(url);
      var dados = await resp.json();
      renderizar(dados);
  }
  ```
- **Frameworks** — React, Vue, Angular usam os mesmos conceitos (estado, componentes, eventos, requisições), só com mais ferramentas em volta. As 3 camadas que você aprendeu (API/UI/Ligação) viram, respectivamente: services, components, hooks/composables.
- **TypeScript** — JavaScript com tipos. Pega erros de digitação **antes** de rodar. Quase obrigatório em qualquer projeto sério hoje.

Você está pronto pra qualquer um desses. 🚀
