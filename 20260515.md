## Bloco 1 — A história do AJAX (45 min)

### O mundo antes do AJAX (1991 – 1998)

Em 1991, Tim Berners-Lee solta a primeira página web no CERN. A web nasceu **estática e síncrona**:

1. Você clica em um link.
2. O navegador manda uma requisição **inteira** ao servidor.
3. O servidor responde com uma página **inteira** nova.
4. O navegador joga fora tudo que tinha e desenha a página nova.

Tudo piscava. Toda interação custava uma página inteira: um cadastro de 8 campos com validação no servidor podia ficar 8 vezes em "tela branca". O usuário **via** a página recarregar.

Você quer ver isso doendo? Pense num formulário de inscrição em 1997: você apertava "Cadastrar", a tela ficava branca por 4 segundos, voltava dizendo "campo CEP inválido", e você tinha que **digitar tudo de novo** porque o servidor não preservou os outros campos.

### O começo do "assíncrono" — Microsoft, 1999

A Microsoft inventou algo chamado **`XMLHTTP`** para o Outlook Web Access (sim, o webmail). A ideia: deixar o JavaScript no navegador conversar com o servidor **sem recarregar a página**.

Em 1999, no Internet Explorer 5, virou um objeto chamado **`XMLHttpRequest`** (XHR para os íntimos). Outros navegadores adotaram. Mas quase ninguém usou — era considerado uma curiosidade técnica.

### A virada de chave — Google, 2004 – 2005

Em 2004 / 2005, o Google solta dois produtos que mudam tudo:

- **Gmail (2004)** — você apaga um e-mail, e a lista atualiza sem piscar.
- **Google Maps (2005)** — você arrasta o mapa, ele carrega os tiles novos enquanto você arrasta. **Sem reload.**

Pela primeira vez, milhões de usuários comuns viram a web se comportar como um **aplicativo de desktop**.

### O batismo — 18 de fevereiro de 2005

Um designer chamado **Jesse James Garrett** escreve um artigo curtinho intitulado:

> *"Ajax: A New Approach to Web Applications"*

Nele, ele dá nome ao bicho:

> **AJAX = Asynchronous JavaScript And XML**

Não era uma tecnologia nova. Era um **nome** para a combinação de tecnologias que o Google já estava usando: `XMLHttpRequest` + JavaScript + DOM. O nome pegou. Pegou tanto que virou substantivo, verbo, adjetivo.

### A explosão — jQuery, 2006

`XMLHttpRequest` puro era um inferno: APIs diferentes em cada navegador, callbacks aninhados, tratamento manual de cada caso de erro. Em janeiro de 2006, um cara chamado **John Resig** lança o **jQuery**.

A promessa do jQuery: *"escreva menos, faça mais"*. Em vez de 25 linhas de XHR, você escrevia:

```javascript
$.get('dados.txt', function(resposta) {
    $('#caixa').html(resposta);
});
```

Em 2010, **mais de 50% dos sites do mundo** usavam jQuery. Era o framework JavaScript do planeta.

### Hoje (2026) — onde estamos

- AJAX como **conceito** está em tudo: Instagram, Twitter, WhatsApp Web, qualquer SPA.
- `XMLHttpRequest` ainda existe, mas hoje em dia se usa mais o **`fetch()`** (API nativa do navegador, mais moderna).
- O **JSON** matou o XML — o "X" do AJAX virou histórico.
- O **jQuery** perdeu espaço para React, Vue, Angular — mas ainda roda em ~70% dos sites do mundo (WordPress, Bootstrap, sistemas legados, intranets).

**Por que ainda aprender jQuery?** Porque:

1. Vai estar em qualquer manutenção de sistema com 5+ anos.
2. É o jeito mais **rápido e didático** de entender AJAX e manipulação de DOM.
3. Os conceitos (seletores, encadeamento, callbacks) são os mesmos que você usará em qualquer framework moderno.

## Bloco 2 — O que é AJAX, de verdade (30 min)

**AJAX não é uma linguagem. AJAX não é um produto. AJAX é uma técnica.**

A técnica consiste em: deixar o JavaScript pedir dados ao servidor **em segundo plano**, e usar esses dados para atualizar **só uma parte** da página, sem recarregar tudo.

### O modelo síncrono (tradicional)

```
Usuário ──► Navegador ──► Servidor
              │
              ▼
       (tela branca enquanto espera)
              │
              ▼
       Página inteira nova
```

### O modelo AJAX (assíncrono)

```
Usuário ──► Navegador ──► Servidor (em segundo plano)
              │              │
              ▼              ▼
       Página segue          Resposta volta
       funcionando           (só os dados)
              ◄───────────────┘
              │
              ▼
       JavaScript atualiza
       só a parte que mudou
```

### As 4 peças do AJAX

1. **HTML** — estrutura da página.
2. **JavaScript** — o cérebro (faz o pedido e usa a resposta).
3. **Um meio de transporte** — `XMLHttpRequest`, `fetch()`, ou `$.ajax()` do jQuery.
4. **Um formato de dados** — antigamente XML, hoje **JSON** quase sempre.

### Síncrono vs assíncrono — analogia

- **Síncrono:** você liga para o restaurante, espera na linha enquanto o atendente confere o estoque, não pode fazer mais nada.
- **Assíncrono:** você pede pelo iFood, fecha o app, continua sua vida; quando chegar, te avisam.

**AJAX é o iFood da web.**

## Bloco 3 — Instalação do jQuery (30 min)

Não existe instalador do jQuery. jQuery é **um arquivo JavaScript**. Você precisa apenas que esse arquivo seja carregado pela página.

Existem **duas formas**.

### Forma 1 — CDN (recomendada para aprender)

CDN = Content Delivery Network. Um servidor público hospeda o jQuery e qualquer pessoa pode usar.

Vantagens: nada para baixar, sempre atualizado, cache compartilhado entre sites.
Desvantagens: precisa de internet.

```html
<script src="https://code.jquery.com/jquery-3.7.1.min.js"></script>
```

### Forma 2 — Local (recomendada para produção)

Você baixa o arquivo `jquery-3.7.1.min.js` de [jquery.com](https://jquery.com) e coloca na pasta do seu projeto.

```html
<script src="js/jquery-3.7.1.min.js"></script>
```

Vantagens: funciona offline, você controla a versão.
Desvantagens: precisa baixar o arquivo.

### Onde colocar o `<script>`?

**Sempre antes do fechamento do `</body>`.** Por quê?

- Se você colocar no `<head>`, o navegador carrega o jQuery **antes** do HTML existir. Aí seu código tenta mexer em elementos que ainda não foram desenhados → bug silencioso.
- Colocando no final do `<body>`, quando o seu script roda, **tudo já está na tela**.

> *(Existe uma solução elegante chamada `$(document).ready(...)` que resolve isso de outro jeito — mas ela usa um **evento**, e a gente combinou que eventos só na semana que vem. Por hoje, **script no final do body** e fim.)*

1. Caminho do `<script>` errado.

3. Erro de digitação (abre o **Console** do navegador: `F12` → aba *Console*).

### Como saber se o jQuery está carregado?

No console do navegador (`F12`):

```javascript

jQuery   // deve mostrar a mesma função
$.fn.jquery  // deve mostrar: "3.7.1"
```

Se der `$ is not defined`, o jQuery **não carregou**.

## Bloco 4 — A função `$()` — o coração do jQuery (30 min)

Tudo no jQuery passa por uma única função: **`$()`**.

`$` é apenas um **apelido**. O nome verdadeiro é `jQuery`. Estas duas linhas fazem **exatamente a mesma coisa**:

```javascript
$('#titulo').text('Olá');
jQuery('#titulo').text('Olá');
```

Usamos `$` porque é mais curto. Só isso. Não tem magia.

### O que `$()` faz?

Ela recebe **algo** e devolve um **objeto jQuery** — uma "caixa" com elementos do DOM dentro e um monte de métodos para mexer neles.

Os 4 tipos de coisa que `$()` aceita:

#### 1. Uma string com seletor CSS — pega elementos existentes

```javascript
$('p')           // todos os parágrafos
$('#menu')       // o elemento de id "menu"
$('.destaque')   // todos os elementos de classe "destaque"
```

#### 2. Uma string com HTML — cria elementos novos

```javascript
$('<div>Olá!</div>')   // cria um <div> na memória (ainda não está na página)
```

#### 3. Um elemento do DOM puro — encapsula em jQuery

```javascript
$(document)
$(document.body)
```

#### 4. Uma função — atalho para "quando o DOM estiver pronto" *(usa evento, pulamos hoje)*

### O conceito mais importante: o objeto jQuery é uma **coleção**

Mesmo quando você seleciona um único elemento, o jQuery devolve uma **coleção** (como uma lista). Quando você chama um método nele, o método age em **todos** os elementos da coleção.

```javascript
$('p').css('color', 'blue');  // pinta TODOS os parágrafos de azul de uma vez
```

Nada de `for`, nada de loop manual. **Esse é o superpoder do jQuery.**

### Exercício 02 — Testando `$()` no console

## Bloco 5 — Seletores: o supermercado do jQuery (60 min)

A regra é: **todo seletor CSS que funciona no navegador, funciona no jQuery**. E o jQuery ainda adiciona **mais seletores próprios**.

Vou listar tudo. Depois a gente pratica.

### 5.1 — Seletores básicos (idênticos ao CSS)

| Seletor          | Pega                              | Exemplo                          |
|------------------|-----------------------------------|----------------------------------|
| `*`              | todos os elementos                | `$('*')`                         |
| `tag`            | todas as tags daquele tipo        | `$('p')`, `$('div')`             |
| `#id`            | elemento de id específico         | `$('#menu')`                     |
| `.classe`        | elementos de uma classe           | `$('.destaque')`                 |
| `tag.classe`     | tag E classe                      | `$('p.destaque')`                |
| `a, b, c`        | união (tudo isso)                 | `$('h1, h2, h3')`                |

### 5.2 — Seletores hierárquicos

| Seletor    | Significa                                       | Exemplo                  |
|------------|-------------------------------------------------|--------------------------|
| `a b`      | `b` descendente de `a` (em qualquer nível)      | `$('ul li')`             |
| `a > b`    | `b` filho **direto** de `a`                     | `$('ul > li')`           |
| `a + b`    | `b` imediatamente depois de `a` (irmão)         | `$('h1 + p')`            |
| `a ~ b`    | `b` irmão de `a` (qualquer depois)              | `$('h1 ~ p')`            |

### 5.3 — Seletores de atributo

| Seletor              | Significa                                  |
|----------------------|--------------------------------------------|
| `[attr]`             | tem o atributo                             |
| `[attr="valor"]`     | atributo igual a "valor"                   |
| `[attr^="valor"]`    | atributo começa com "valor"                |
| `[attr$="valor"]`    | atributo termina com "valor"               |
| `[attr*="valor"]`    | atributo contém "valor"                    |

Exemplos:

```javascript
$('input[type="text"]')         // só inputs de texto
$('a[href^="https"]')           // links que começam com https
$('img[src$=".png"]')           // imagens PNG
$('a[href*="etec"]')            // links que contêm "etec"
```

### 5.4 — Filtros jQuery (esses não são CSS!)

| Filtro            | Pega                                      |
|-------------------|-------------------------------------------|
| `:first`          | o primeiro elemento da coleção            |
| `:last`           | o último                                  |
| `:even`           | os de índice par (0, 2, 4...)             |
| `:odd`            | os de índice ímpar (1, 3, 5...)           |
| `:eq(n)`          | o elemento de índice `n`                  |
| `:gt(n)`          | elementos com índice maior que `n`        |
| `:lt(n)`          | elementos com índice menor que `n`        |
| `:contains("x")`  | elementos que contêm o texto "x"          |
| `:empty`          | elementos sem nenhum filho                |
| `:has(seletor)`   | elementos que contêm o seletor dentro     |
| `:not(seletor)`   | elementos que **não** batem com o seletor |
| `:hidden`         | elementos escondidos                      |
| `:visible`        | elementos visíveis                        |

Exemplos:

```javascript
$('li:first')              // primeira <li>
$('tr:odd')                // linhas ímpares de tabela (zebra striping!)
$('p:contains("AJAX")')    // parágrafos que mencionam AJAX
$('div:has(img)')          // divs que têm imagem dentro
$('input:not([disabled])') // inputs que não estão desabilitados
```

### 5.5 — Filtros de formulário

| Filtro       | Pega                                |
|--------------|-------------------------------------|
| `:input`     | qualquer campo (`input`, `textarea`, `select`, `button`) |
| `:text`      | só inputs de texto                  |
| `:password`  | inputs de senha                     |
| `:checkbox`  | checkboxes                          |
| `:radio`     | radios                              |
| `:checked`   | os que estão marcados               |
| `:selected`  | options selecionadas                |
| `:disabled`  | desabilitados                       |
| `:enabled`   | habilitados                         |

### 5.6 — Travessia (não é "seletor", mas vive junto)

A partir de uma coleção, você pode "andar" pela árvore:

| Método          | Pega                                            |
|-----------------|-------------------------------------------------|
| `.find(s)`      | descendentes que casam com `s`                  |
| `.children(s)`  | só filhos diretos                               |
| `.parent()`     | o pai direto                                    |
| `.parents(s)`   | todos os ancestrais (até `<html>`)              |
| `.closest(s)`   | ancestral mais próximo que casa com `s`         |
| `.siblings(s)`  | irmãos                                          |
| `.next()`       | próximo irmão                                   |
| `.prev()`       | irmão anterior                                  |
| `.first()`      | primeiro da coleção                             |
| `.last()`       | último da coleção                               |
| `.eq(n)`        | o de índice `n`                                 |
| `.filter(s)`    | filtra a coleção                                |

```html
    <p>Links externos:</p>
    <ul>
        <li><a href="https://jquery.com">jQuery oficial</a></li>
        <li><a href="https://www.etec.sp.gov.br">ETEC</a></li>
        <li><a href="/local.html">página local</a></li>

        // 1. Zebra-striping: linhas ímpares com fundo cinza claro
        $('#notas tr:odd').css('background', '#eee');

        // 2. Cabeçalho azulado
        $('#notas tr:first').css('background', '#cfe').css('font-weight', 'bold');

        // 3. Links externos (que começam com https) ficam vermelhos

        // 4. Linhas que contêm a palavra "Felipe" ficam destacadas
        $('tr:contains("Felipe")').css('background', '#fcc');

        // 5. Todas as células de nota (segunda coluna)
        $('#notas tr td:last-child').css('text-align', 'right');
    </script>
</body>
</html>
```

**Brinque:**
- Troque `:odd` por `:even`. O que muda?
- Troque `:contains("Felipe")` por `:contains("a")`. Quantas linhas pegam?
- Troque `a[href^="https"]` por `a[href*="etec"]`. Só um link fica vermelho.

## Bloco 6 — Manipulando o DOM (sem eventos!) (30 min)

Selecionar é metade do trabalho. A outra metade é **mexer** no que você selecionou.

### Conteúdo

| Método           | Faz                                            |
|------------------|------------------------------------------------|
| `.text()`        | lê / escreve o texto puro                      |
| `.text("x")`     | substitui o texto                              |
| `.html()`        | lê / escreve HTML interno                      |
| `.html("<b>x</b>")` | substitui o HTML interno                   |
| `.val()`         | lê / escreve o valor de um campo de formulário |

### Atributos e CSS

| Método                           | Faz                                  |
|----------------------------------|--------------------------------------|
| `.attr("href")`                  | lê o atributo                        |
| `.attr("href", "/novo")`         | escreve o atributo                   |
| `.removeAttr("disabled")`        | remove o atributo                    |
| `.css("color")`                  | lê CSS computado                     |
| `.css("color", "red")`           | escreve CSS                          |
| `.css({color:"red", fontSize:"20px"})` | escreve vários de uma vez     |
| `.addClass("ativo")`             | adiciona classe                      |
| `.removeClass("ativo")`          | remove classe                        |
| `.toggleClass("ativo")`          | inverte: tem? tira. não tem? coloca. |
| `.hasClass("ativo")`             | devolve `true`/`false`               |

### Inserção e remoção

| Método              | Faz                                              |
|---------------------|--------------------------------------------------|
| `.append(x)`        | insere `x` no **final** de cada elemento         |
| `.prepend(x)`       | insere `x` no **início** de cada elemento        |
| `.after(x)`         | insere `x` **depois** do elemento (irmão)        |
| `.before(x)`        | insere `x` **antes** do elemento (irmão)         |
| `.remove()`         | apaga os elementos do DOM                        |
| `.empty()`          | esvazia o conteúdo (mantém o elemento)           |
| `.replaceWith(x)`   | substitui o elemento inteiro por `x`             |

### Efeitos visuais (sem clicar em nada!)

| Método              | Faz                                  |
|---------------------|--------------------------------------|
| `.show()`           | mostra                               |
| `.hide()`           | esconde (`display: none`)            |
| `.fadeIn(ms)`       | aparece com fade                     |
| `.fadeOut(ms)`      | desaparece com fade                  |
| `.slideDown(ms)`    | desce deslizando                     |
| `.slideUp(ms)`      | sobe deslizando                      |
| `.fadeToggle(ms)`   | aparece/some                         |
| `.slideToggle(ms)`  | sobe/desce                           |

### Encadeamento (chaining)

Quase todo método jQuery devolve a **própria coleção** — o que permite **encadear** chamadas:

```javascript
$('#titulo')
    .text('Novo título')
    .css('color', 'crimson')
    .addClass('grande')
    .fadeIn(1000);
```

Lê-se: pegue o `#titulo`, mude o texto, pinte de vermelho, adicione a classe `grande`, e faça aparecer com fade de 1 segundo. **Tudo no mesmo objeto.**

## Bloco 7 — AJAX na prática (45 min)

Chegou a hora. Vamos finalmente **buscar dados do servidor** com AJAX.

### As 3 formas no jQuery

| Método              | Para que serve                              |
|---------------------|---------------------------------------------|
| `$.get(url, cb)`    | atalho para GET (buscar dados)              |
| `$.post(url, dados, cb)` | atalho para POST (enviar dados)         |
| `$.ajax({...})`     | versão completa, com todas as opções        |
| `.load(url)`        | atalho que joga HTML direto num elemento    |
| `$.getJSON(url, cb)`| atalho para GET de JSON                     |

### O conceito de **callback**

Quando você faz `$.get`, o jQuery **não** te devolve a resposta de imediato. Ele não pode — a resposta ainda está vindo pela internet. Em vez disso, você passa uma **função** que será chamada **quando a resposta chegar**.

```javascript
$.get('arquivo.txt', function(resposta) {
    // este código roda DEPOIS, quando a resposta chega
    console.log('chegou:', resposta);
});
console.log('isto roda ANTES de a resposta chegar');
```

A ordem no console será:

```
isto roda ANTES de a resposta chegar
chegou: ...
```

Isso é a **assincronia** do AJAX.

> *Importante: callback não é a mesma coisa que evento. Callback é uma função que você passa para outra função chamar depois. É a estrutura básica de toda chamada AJAX, mesmo sem nenhum evento de usuário envolvido.*

### A API que vamos usar — **JSONPlaceholder**

`https://jsonplaceholder.typicode.com` é uma API pública, gratuita, sem cadastro, que devolve dados falsos de teste — perfeita para aprender AJAX.

Endpoints úteis:

| URL                                                      | Devolve                  |
|----------------------------------------------------------|--------------------------|
| `https://jsonplaceholder.typicode.com/posts`             | lista de 100 posts       |
| `https://jsonplaceholder.typicode.com/posts/1`           | um post específico       |
| `https://jsonplaceholder.typicode.com/users`             | 10 usuários              |
| `https://jsonplaceholder.typicode.com/users/3`           | usuário 3                |
| `https://jsonplaceholder.typicode.com/comments`          | 500 comentários          |
| `https://jsonplaceholder.typicode.com/todos?userId=1`    | tarefas do usuário 1     |

## Bloco 8 — Discussão final: AJAX é bom mesmo? (10 min)

Voltamos à pergunta que dá título a esta seção. Depois de 4 horas, vocês têm ferramentas para opinar.

### O que AJAX te dá

- **UX muito melhor:** sem tela branca, sem perder o estado da página.
- **Menos tráfego:** você baixa só os dados, não o HTML inteiro toda vez.
- **Sensação de aplicativo:** Gmail, Trello, Spotify, WhatsApp Web — todos seriam impossíveis sem AJAX.
- **Separação de preocupações:** o servidor cuida dos dados, o JavaScript cuida da apresentação.

### O que AJAX te cobra

- **Complexidade.** Páginas estáticas são triviais; páginas com AJAX exigem entender assincronia, callbacks, estados de carregamento, tratamento de erro.
- **SEO.** Buscadores indexam o HTML que vem do servidor. Se o conteúdo aparece via AJAX **depois**, o Google pode não enxergar (hoje em dia já enxerga melhor, mas continua sendo cuidado).
- **Acessibilidade.** Leitores de tela esperam ver a página inteira; conteúdo que aparece em segundo plano pode ser ignorado se não for feito direito.
- **Histórico do navegador.** O botão "voltar" não funciona naturalmente em AJAX — você precisa programar isso (History API).
- **Dependência total de JavaScript.** Sem JS, a página não funciona. Em 2026 isso é menos problema, mas em conexões ruins ou navegadores antigos…
- **Estados de erro.** Síncrono: deu errado, mostra erro. AJAX: pode dar timeout, falha de rede, resposta inválida, 500 do servidor — você tem que tratar **todos**.

### A resposta honesta

**AJAX é uma ferramenta.** Como toda ferramenta:

- Página de portfólio estática? Não precisa de AJAX. Não use.
- Sistema de cadastro com 100 telas? Não precisa de AJAX puro hoje — provavelmente você quer um framework (React, Vue, Angular).
- Página que precisa atualizar uma coisinha sem recarregar (um contador, um menu, uma busca instantânea)? **AJAX é perfeito.**
- Aplicação web grande? AJAX é a **base** sobre a qual frameworks modernos são construídos.

**Por isso aprendemos AJAX, mesmo em 2026:** porque ele continua sendo o motor por baixo de tudo que parece "mágico" na web moderna.

### Próxima semana — eventos

Repare que tudo o que fizemos hoje aconteceu **sozinho**, quando a página carregou. Ainda não fizemos a parte mais legal: **reagir ao usuário**.

> *Clicou no botão → busca dados.*
> *Digitou no campo → autocompleta.*
> *Passou o mouse → mostra dica.*

Isso é o mundo dos **eventos** — e é o que vamos atacar na próxima aula.
